# Day02 - 질의 응답

## 🔧 ORA-12541 오류 해결 순서

---

### ✅ 1단계. 리스너(listener)가 실행 중인지 확인

### ⬛ Windows:

1. **명령 프롬프트(cmd)** 실행 (관리자 권한 권장)
2. 아래 명령어 입력:
    
    ```bash
    lsnrctl status
    ```
    
    - 결과에 `STATUS of the LISTENER`가 나오면 → **리스너 실행 중**
    - `TNS-12541` 오류와 함께 나오면 → **리스너 실행 안 됨**

### ⬛ 리스너 수동 시작 (Windows)

```bash
lsnrctl start
```

---

### ✅ 2단계. 리스너가 사용하는 포트 확인 (기본: 1521)

1. `listener.ora` 파일 위치:
    
    ```
    C:\oracle\product\...\network\admin\listener.ora
    ```
    
2. 내부에 이런 내용이 있어야 정상입니다:
    
    ```
    LISTENER =
      (DESCRIPTION_LIST =
        (DESCRIPTION =
          (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
        )
      )
    ```
    

---

### ✅ 3단계. 데이터베이스 인스턴스가 리스너에 등록되었는지 확인

리스너는 실행 중인데도 접속 오류가 발생하면 **데이터베이스가 리스너에 등록되지 않은 상태**일 수 있습니다.

### 🔸 수동 등록:

1. **SQL*Plus**에서 SYS/SYSDBA로 접속:
    
    ```bash
    sqlplus / as sysdba
    ```
    
2. 리스너에 인스턴스 등록 명령:
    
    ```sql
    alter system register;
    ```
    

---

### ✅ 4단계. SQL Developer에서 접속 정보 재확인

| 항목 | 확인 |
| --- | --- |
| **Host** | `localhost` 또는 IP 주소 |
| **Port** | `1521` |
| **Service name** | 설치 시 지정한 이름 (예: `XE`, `FREE`, `ORCL`) |

> tnsnames.ora 없이 Basic 모드 접속이 가장 쉬움
> 

---

## 🧪 테스트용 연결 확인 방법 (SQL*Plus 또는 터미널)

```bash
tnsping XE
```

또는

```bash
tnsping localhost:1521/XE
```

## 🆘 여전히 안 될 경우 확인할 것

| 점검 항목 | 설명 |
| --- | --- |
| `services.msc` | Oracle Listener 서비스가 "실행 중"인지 확인 |
| 방화벽 | 포트 1521이 열려 있는지 확인 |
| 설치된 Oracle 버전 | Oracle 23c Free, XE 등 설치가 올바른지 확인 |

---

Oracle SQL Developer에서 정상 접속을 위해 필요한 **listener.ora**, **tnsnames.ora**, **sqlnet.ora**의 설정 예시입니다. 이 예시는 보통 Oracle 18c XE 또는 Oracle 23c Free (Non-CDB 기반)의 **로컬 PC 접속** 환경에 적합합니다.

---

## ✅ 1. `listener.ora` 예시

📁 경로:

```
C:\oracle\product\{버전}\dbhomeXE\network\admin\listener.ora
```

```

LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    )
  )

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (SID_NAME = XE)  -- 또는 FREE, ORCL 등 설치된 서비스 이름
      (ORACLE_HOME = C:\oracle\product\{버전}\dbhomeXE)
    )
  )
```

> ✅ Oracle 23c Free를 설치한 경우 SID_NAME = FREE 일 수 있음.
> 
> 
> ✅ Oracle이 제대로 설치되었으면 `lsnrctl status` 명령어로 확인 가능.
> 

---

## ✅ 2. `tnsnames.ora` 예시

📁 경로:

```

C:\oracle\product\{버전}\dbhomeXE\network\admin\tnsnames.ora
```

```

XE =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = XE)  -- 또는 FREE, ORCL 등
    )
  )

```

> ✅ SERVICE_NAME은 DB 생성 시 지정한 이름입니다.
> 
> 
> 기본값: XE (18c XE), FREE (23c Free), ORCL (19c 등)
> 

---

## ✅ 3. `sqlnet.ora` 예시

📁 경로:

```

C:\oracle\product\{버전}\dbhomeXE\network\admin\sqlnet.ora
```

```

SQLNET.AUTHENTICATION_SERVICES = (NTS)
NAMES.DIRECTORY_PATH = (TNSNAMES, EZCONNECT)
```

> ✅ NTS는 Windows 인증 연동을 의미합니다.
> 
> 
> ✅ `TNSNAMES`가 먼저 오도록 설정해야 tnsnames.ora를 우선 사용합니다.
> 

---

## ✅ Oracle SQL Developer에서 접속 설정 방법

| 항목 | 입력값 |
| --- | --- |
| **Connection Name** | MyXE 또는 원하는 이름 |
| **Username** | scott 또는 sys |
| **Password** | 설치 시 설정한 비밀번호 |
| **Role** | default (SCOTT 계정), SYSDBA (SYS 계정) |
| **Connection Type** | `TNS` 선택 |
| **Network Alias** | `XE` ← tnsnames.ora에서 설정한 이름과 일치해야 함 |

> 또는 Connection Type = Basic으로 하고 직접 Host, Port, Service Name을 입력해도 됩니다.
>

## ✅ 문제 해결 팁

| 문제 | 해결 방법 |
| --- | --- |
| 리스너가 작동하지 않음 | `lsnrctl start` 명령 실행 |
| 포트 1521 차단 | 방화벽에서 예외 허용 |
| tnsnames.ora 무시됨 | SQL Developer에서 `TNS` 대신 `Basic`으로 접속 시도 |
| 서비스 이름 오류 | `lsnrctl status`로 실제 등록된 SERVICE_NAME 확인 |

C:\app\oracle\product\23c\dbhomeXE\network\admin\listener.ora

```sql
# listener.ora Network Configuration File: C:\app\ASUS\product\23ai\dbhomeFree\NETWORK\ADMIN\listener.ora
# Generated by Oracle configuration tools.

DEFAULT_SERVICE_LISTENER = FREE

LISTENER =

    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 192.168.0.174)(PORT = 1521))
      (ADDRESS = (PROTOCOL = IPC)(KEY = EXTPROC1521))
    )
  )

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = FREE)
      (SID_NAME = FREE)

    )
  )
```
