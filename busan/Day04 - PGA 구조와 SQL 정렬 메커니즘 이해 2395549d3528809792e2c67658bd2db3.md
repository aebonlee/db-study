## 📘 PGA 구조와 SQL 정렬 메커니즘

### 1. PGA란?

- **Program Global Area**
- 각 Server Process마다 **개별로 할당**되는 메모리 구조
- 정렬(Sort), 해싱(Hash), 커서 관리, 바인드 변수 등 다양한 연산 처리에 사용

### 2. SQL 정렬에 사용되는 PGA 구성 요소

- **Sort Area**: `ORDER BY`, `GROUP BY`, `DISTINCT` 등 정렬 처리
- **Hash Area**: `HASH JOIN` 등 해시 기반 조인 처리

### 3. PGA 부족 시 처리 방식

- PGA 메모리가 부족하면 정렬/해싱 데이터를 디스크에 저장 (→ Temp Tablespace 사용)
- 이로 인해 `direct path write temp`, `direct path read temp` Wait Event 발생

### ⚠️ [확인된 실습 누락 및 오류 사항]

### ✅ 문제 1. 실습 SQL 누락

- 교재 내 `PGA 정렬 메커니즘`에 대한 설명은 **이론 중심**으로 되어 있으며, 다음과 같은 시나리오만 도식화되어 있음:
    - SELECT * FROM CUSTOMER ORDER BY CUSTOMER_NAME DESC
    - 정렬 대상이 크면 PGA → Temp Tablespace 이동 흐름 설명
- 하지만 정렬 SQL 수행 시 Temp 사용 여부를 실습으로 검증하는 SQL 예제(`v$tempseg_usage` 확인)는 누락되어 있음.

### ✅ 문제 2. Temp 사용 유도를 위한 `PGA 설정 수동화` 실습 누락

- 수동 설정 예시는 다음처럼 서술되어 있음:
    
    ```sql
    ALTER SESSION SET WORKAREA_SIZE_POLICY=MANUAL;
    ALTER SESSION SET SORT_AREA_SIZE=800M;
    ```
    
- 하지만 이것을 사용해 **정렬 쿼리를 실행**하고, Temp 사용 여부를 확인하는 **전체 실습 흐름**이 포함되어 있지 않음.

### ✅ 문제 3. 실습 후 리소스 모니터링 SQL 없음

- 아래 항목이 있어야 실습 효과를 검증할 수 있으나 누락됨:
    
    ```sql
    SELECT * FROM v$tempseg_usage;
    SELECT * FROM v$sesstat WHERE name LIKE '%PGA%';
    ```
    

---

### ✅ 보완을 위한 제안 실습 구성

```sql
-- 1. 세션별 PGA 설정 수동화
ALTER SESSION SET WORKAREA_SIZE_POLICY = MANUAL;
ALTER SESSION SET SORT_AREA_SIZE = 100000;

-- 2. 정렬 대상 테이블 생성
CREATE TABLE sort_test AS
SELECT LEVEL AS id, DBMS_RANDOM.STRING('U', 10) AS name
FROM dual CONNECT BY LEVEL <= 100000;

-- 3. 정렬 수행
SELECT * FROM sort_test ORDER BY name;

-- 4. TEMP 사용 여부 확인
SELECT username, tablespace, blocks, segtype
FROM v$tempseg_usage;

-- 5. PGA 사용량 확인
SELECT name, value
FROM v$sesstat s, v$statname n
WHERE s.statistic# = n.statistic#
AND s.sid IN (SELECT sid FROM v$mystat WHERE ROWNUM = 1)
AND name LIKE '%session PGA%';
```

## 🧪 실습 2: 정렬(SQL Sort) 테스트

### 1. 테스트 테이블 생성 및 데이터 삽입

```sql
CREATE TABLE sort_test (
    id NUMBER,
    name VARCHAR2(100)
);

BEGIN
  FOR i IN 1..100000 LOOP
    INSERT INTO sort_test VALUES (i, '고객_' || DBMS_RANDOM.STRING('A', 10));
  END LOOP;
  COMMIT;
END;
```

### 2. 정렬 수행 및 실행계획 확인

```sql
-- Sort 발생 SQL
EXPLAIN PLAN FOR
SELECT * FROM sort_test ORDER BY name;

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

## 🧪 실습 4: Hash Join과 Nested Loop 비교

```sql
-- Hash Join 유도 (PGA 사용)
SELECT /*+ USE_HASH(a b) */ a.id, b.name
FROM sort_test a JOIN sort_test b ON a.id = b.id;

-- Nested Loop Join 유도 (PGA 거의 사용 안 함)
SELECT /*+ USE_NL(a b) */ a.id, b.name
FROM sort_test a JOIN sort_test b ON a.id = b.id;
```
