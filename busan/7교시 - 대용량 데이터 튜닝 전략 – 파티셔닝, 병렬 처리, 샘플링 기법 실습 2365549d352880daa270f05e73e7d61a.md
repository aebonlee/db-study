# 7교시 - 대용량 데이터 튜닝 전략 – 파티셔닝, 병렬 처리, 샘플링 기법 실습

### 🎯 학습 목표

- Oracle에서 지원하는 파티셔닝(PARTITIONING)의 개념과 동작 방식을 이해한다.
- 병렬 처리(PARALLEL QUERY)의 활용 시기와 실행계획 상의 변화 포인트를 분석한다.
- 대용량 테이블에서 통계 수집 및 샘플링 전략을 통해 효율적인 SQL 튜닝을 수행할 수 있다.

### 🧑‍🏫 강의 흐름 및 설명 내용

    이번 시간에는 Oracle이 제공하는 대용량 처리 기술인 **파티셔닝**, **병렬 처리**, 그리고 **샘플링 기반의 통계 전략**에 대해 학습하겠습니다.”

1. **[파티셔닝 개념 및 종류 설명 – 10분]**
    - 파티셔닝이란?
        
        → 하나의 대용량 테이블을 논리적으로 나누어 관리함으로써 **접근 범위 축소**, **I/O 분산**, **백업/복구 단순화** 등을 가능하게 하는 기술
        
    - 주요 유형:
        - **Range Partition** – 날짜/숫자 범위 기반
        - **List Partition** – 특정 값 기준 (예: 지역 코드)
        - **Hash Partition** – 균등 분산
        - **Composite Partition** – 위 방법들을 혼합
    - 파티션 프루닝(Partition Pruning):
        
        → WHERE 조건절에 따라 접근할 파티션만 선택 → I/O 절감
        

---

1. **[실습 – 파티셔닝 테이블 생성 및 SQL 실행계획 비교 – 10분]**
    
    🔸 파티션 테이블 생성
    
    ```sql
    
    CREATE TABLE SALES_PART (
      SALE_ID NUMBER,
      CUST_ID NUMBER,
      AMOUNT NUMBER,
      SALE_DATE DATE
    )
    PARTITION BY RANGE (SALE_DATE) (
      PARTITION P2022 VALUES LESS THAN (TO_DATE('2023-01-01','YYYY-MM-DD')),
      PARTITION P2023 VALUES LESS THAN (TO_DATE('2024-01-01','YYYY-MM-DD')),
      PARTITION P2024 VALUES LESS THAN (MAXVALUE)
    );
    ```
    
    🔸 파티션 조회 확인
    
    ```sql
    
    SELECT TABLE_NAME, PARTITION_NAME
    FROM USER_TAB_PARTITIONS
    WHERE TABLE_NAME = 'SALES_PART';
    ```
    
    🔸 파티션 프루닝 SQL 실행
    
    ```sql
    
    EXPLAIN PLAN FOR
    SELECT * FROM SALES_PART WHERE SALE_DATE = TO_DATE('2023-07-01','YYYY-MM-DD');
    SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
    
    ```
    
    ➤ 실행계획에서 `PARTITION RANGE SINGLE` 확인
    
    ➤ 전체 스캔 대비 I/O 최소화 효과 설명
    

---

1. **[병렬 처리(PARALLEL QUERY) 설명 및 실습 – 10분]**
    - 병렬 처리란?
        
        → SQL 실행 시 여러 프로세스가 동시에 작업을 분산하여 처리 (Slave Process)
        
    - 병렬 힌트 사용:
        
        ```sql
        
        SELECT /*+ PARALLEL(SALES_PART 4) */ * FROM SALES_PART;
        ```
        
    - 실행계획 확인 시 `PX`(Parallel Execution), `Table Access Full`, `PX BLOCK` 등 표시
    
    ➤ 실무에서는 OLAP, 배치성 작업, 통계 처리, 대량 DML에서 주로 사용
    
    ➤ OLTP 환경에서는 주의 필요 (락 경합, 자원 고갈 유발 가능)
    

---

1. **[샘플링 기법을 활용한 통계 수집 전략 – 5분]**
    - `DBMS_STATS.GATHER_TABLE_STATS` 사용 시
        
        **AUTO_SAMPLE_SIZE**, 또는 퍼센트 설정으로 전체 테이블 스캔 방지 가능
        
    
    ```sql
    
    BEGIN
      DBMS_STATS.GATHER_TABLE_STATS(
        ownname => USER,
        tabname => 'SALES_PART',
        estimate_percent => 10,
        cascade => TRUE
      );
    END;
    ```
    
    ➤ 대용량 테이블에서 전체 통계 수집은 비효율
    
    ➤ 샘플링 수집 후 실행계획에 반영된 결과 해석
    

---

1. **[정리 및 마무리 – 5분]**
    - 대용량 테이블에서 가장 중요한 것은 ‘데이터 접근 범위’를 줄이는 전략
    - 파티셔닝은 물리적 I/O를 줄이는 핵심 기술
    - 병렬 처리는 속도를 높이는 유용한 도구지만, 잘못 쓰면 자원 경합 유발
    - “대용량 데이터 튜닝은 정밀한 외과 수술이 아니라, **시스템 설계를 바꾸는 건축 작업**이다.”
