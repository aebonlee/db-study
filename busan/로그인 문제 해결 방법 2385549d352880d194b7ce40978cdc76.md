## ✅ 1. **비밀번호 오류 (ORA-01017)**

> ❗ 오류 메시지: ORA-01017: invalid username/password; logon denied
> 

### ✅ 해결 방법:

- **대소문자 구분** 주의: Oracle은 **비밀번호 대소문자를 구분**합니다.
- **계정 잠김 확인**:
    1. SQL*Plus 또는 SQL Developer에서 SYS 계정으로 접속 시도 (`Role: SYSDBA`)
    2. 아래 명령어 실행:
        
        ```sql
        SELECT username, account_status FROM dba_users WHERE username = 'SCOTT';
        ```
        
    3. 계정이 `LOCKED` 상태면 아래 명령으로 해제:
        
        ```sql
        ALTER USER SCOTT ACCOUNT UNLOCK;
        ALTER USER SCOTT IDENTIFIED BY oracle123!; -- 비밀번호 재설정
        ```
        

---

## ✅ 2. **호스트, 포트, 서비스 이름 오류 (ORA-12514, ORA-12170 등)**

### ✅ 확인 사항:

| 항목 | 점검 내용 |
| --- | --- |
| Hostname | `localhost` 또는 실제 IP 맞는지 확인 |
| Port | 기본은 `1521` – 방화벽에서 열려 있어야 함 |
| Service name | `XE`, `FREE`, `ORCL` 등 정확히 입력해야 함 |
| Listener 상태 | 명령어: `lsnrctl status` 로 Listener 구동 상태 확인 |
| DB가 켜져 있는지 | Windows에서는 `services.msc`에서 Oracle 서비스 확인 |

> 예: Oracle 23c Free인 경우 서비스 이름은 FREE입니다.
> 

---

## ✅ 3. **SYS 계정 로그인 시 Role 설정**

> Role을 기본(default)으로 설정하면 오류 발생.

## 🔧 ORA-12541 오류 해결 순서

### ✅ 1단계. 리스너(listener)가 실행 중인지 확인

### ⬛ Windows:

1. **명령 프롬프트(cmd)** 실행 (관리자 권한 권장)
2. 아래 명령어 입력:
    
    ```bash
    lsnrctl status
    ```
    
    - 결과에 `STATUS of the LISTENER`가 나오면 → **리스너 실행 중**
    - `TNS-12541` 오류와 함께 나오면 → **리스너 실행 안 됨**

### ⬛ 리스너 수동 시작 (Windows)

```bash
lsnrctl start
```

---

### ✅ 2단계. 리스너가 사용하는 포트 확인 (기본: 1521)

1. `listener.ora` 파일 위치:
    
    ```
    C:\oracle\product\...\network\admin\listener.ora
    ```
    
2. 내부에 이런 내용이 있어야 정상입니다:
    
    ```
    
    LISTENER =
      (DESCRIPTION_LIST =
        (DESCRIPTION =
          (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
        )
      )
    
    ```
    

---

### ✅ 3단계. 데이터베이스 인스턴스가 리스너에 등록되었는지 확인

리스너는 실행 중인데도 접속 오류가 발생하면 **데이터베이스가 리스너에 등록되지 않은 상태**일 수 있습니다.

### 🔸 수동 등록:

1. **SQL*Plus**에서 SYS/SYSDBA로 접속:
    
    ```bash
    sqlplus / as sysdba
    ```
    
2. 리스너에 인스턴스 등록 명령:
    
    ```sql
    alter system register
    ```
    

---

### ✅ 4단계. SQL Developer에서 접속 정보 재확인

| 항목 | 확인 |
| --- | --- |
| **Host** | `localhost` 또는 IP 주소 |
| **Port** | `1521` |
| **Service name** | 설치 시 지정한 이름 (예: `XE`, `FREE`, `ORCL`) |

> tnsnames.ora 없이 Basic 모드 접속이 가장 쉬움
>

## ✅ 1. `listener.ora` 예시

📁 경로:

```
C:\oracle\product\{버전}\dbhomeXE\network\admin\listener.ora
```

```
ISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    )
  )

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (SID_NAME = XE)  -- 또는 FREE, ORCL 등 설치된 서비스 이름
      (ORACLE_HOME = C:\oracle\product\{버전}\dbhomeXE)
    )
  )
```

> ✅ Oracle 23c Free를 설치한 경우 SID_NAME = FREE 일 수 있음.
> 
> 
> ✅ Oracle이 제대로 설치되었으면 `lsnrctl status` 명령어로 확인 가능.
>

## ✅ 2. `tnsnames.ora` 예시

📁 경로:

```
C:\oracle\product\{버전}\dbhomeXE\network\admin\tnsnames.ora
```

```

XE =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    (CONNECT_DATA =
      (SERVER = DEDICATED)
      (SERVICE_NAME = XE)  -- 또는 FREE, ORCL 등
    )
  )
```

> ✅ SERVICE_NAME은 DB 생성 시 지정한 이름입니다.
> 
> 
> 기본값: XE (18c XE), FREE (23c Free), ORCL (19c 등)
>

## ✅ 해결 방법: SCOTT 계정 수동 생성

Oracle에서는 SCOTT 계정을 수동으로 생성해야 합니다. 아래 절차를 따라 주세요.

---

### 🔧 1단계. SQL*Plus에서 SYSDBA로 접속 (이미 접속되어 있으면 생략)

```bash
sqlplus / as sysdba
```

---

### 🔧 2단계. SCOTT 사용자 및 스키마 생성

```sql
-- 사용자 생성
CREATE USER scott IDENTIFIED BY oracle123!;

-- 권한 부여
GRANT CONNECT, RESOURCE TO scott;

-- 테이블스페이스 설정 (옵션)
ALTER USER scott DEFAULT TABLESPACE users;
ALTER USER scott TEMPORARY TABLESPACE temp;
```

---

### 🔧 3단계. 테스트 테이블 생성 (선택)

```sql
CONN scott/oracle1@localhost:1521/FREE

CREATE TABLE emp (
  empno NUMBER(4) PRIMARY KEY,
  ename VARCHAR2(10),
  job VARCHAR2(9),
  mgr NUMBER(4),
  hiredate DATE,
  sal NUMBER(7, 2),
  comm NUMBER(7, 2),
  deptno NUMBER(2)
);
```

## 🎯 해결 절차 (SQL*Plus에서 정확히 입력하세요)

### ① SQL*Plus 접속

## ✅ 점검 1: SCOTT 계정 잠김 여부 확인

SQL*Plus에서 다음 명령을 실행하세요:

```sql
SELECT username, account_status FROM dba_users WHERE username = 'SCOTT';
```

정상 출력 예:

```
USERNAME   ACCOUNT_STATUS
---------  ---------------
SCOTT      OPEN
```

→ 만약 `LOCKED` 또는 `EXPIRED`이면 다음 실행:

```sql
ALTER USER scott ACCOUNT UNLOCK;
ALTER USER scott IDENTIFIED BY welcome1;
```

```sql
GRANT CONNECT, RESOURCE TO scott;
```
