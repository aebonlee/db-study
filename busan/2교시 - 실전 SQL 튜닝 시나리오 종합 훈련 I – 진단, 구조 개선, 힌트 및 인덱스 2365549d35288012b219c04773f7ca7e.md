1. **[문제 시나리오 SQL 제공 – 5분]**
    
    ▶ 실습용 SQL 예제:
    
    ```sql
    SELECT O.ORDER_ID, O.ORDER_DATE, C.CUST_NAME, P.PROD_NAME
    FROM ORDERS O, CUSTOMERS C, PRODUCTS P
    WHERE O.CUST_ID = C.CUST_ID
      AND O.PROD_ID = P.PROD_ID
      AND TO_CHAR(O.ORDER_DATE, 'YYYY-MM') = '2023-01'
      AND P.CATEGORY LIKE '%전자%'
      AND C.STATUS = 'SUSPENDED';
    ```
    
    ▶ 예상 문제점:
    
    - TO_CHAR → 인덱스 무력화
    - LIKE '%전자%' → Prefix Index 무효
    - 조인 순서 → 옵티마이저 판단에만 의존
    - 통계 부정확 → 계획 오판 가능성

---

1. **[실행계획 분석 및 병목 진단 – 10분]**
    
    ▶ `EXPLAIN PLAN FOR` 또는 `DBMS_XPLAN.DISPLAY` 실행
    
    ▶ 병목 진단 포인트:
    
    | 요소 | 관찰 내용 | 개선 포인트 |
    | --- | --- | --- |
    | TABLE ACCESS FULL | ORDERS, PRODUCTS | 조건절 개선 필요 |
    | FILTER | TO_CHAR 사용 | 제거 → 날짜 범위 조건으로 |
    | INDEX SKIP SCAN or NOT USED | 없음 | 인덱스 설계 필요 |
    | COST 집중 | 특정 조인 단계 | 조인 순서 재설계 or 힌트 |

---

1. **[개선 전략 설계 및 적용 – 15분]**
    
    ▶ 구조 변경:
    
    ```sql
    WHERE O.ORDER_DATE BETWEEN TO_DATE('2023-01-01','YYYY-MM-DD')
                           AND TO_DATE('2023-01-31','YYYY-MM-DD')
      AND P.CATEGORY LIKE '전자%'
    ```
    
    ▶ 인덱스 생성:
    
    ```sql
    CREATE INDEX IDX_ORDERS_ODATE ON ORDERS(ORDER_DATE);
    CREATE INDEX IDX_PRODUCTS_CAT ON PRODUCTS(CATEGORY);
    ```
    
    ▶ 힌트 적용:
    
    ```sql
    SELECT /*+ LEADING(O C P) USE_NL(P) */ ...
    ```
    
    ▶ 통계 수집:
    
    ```sql
    EXEC DBMS_STATS.GATHER_TABLE_STATS(USER, 'ORDERS');
    ```
    

---

1. **[튜닝 전후 실행계획 및 수치 비교 – 10분]**
    
    
    | 지표 | 튜닝 전 | 튜닝 후 |
    | --- | --- | --- |
    | COST | 360 | 48 |
    | BUFFER GETS | 25,000 | 2,100 |
    | DISK READS | 3,800 | 320 |
    | ROWS | 1,000 | 1,000 (동일) |
    
    ▶ 실행계획 변화 확인:
    
    - TABLE ACCESS FULL → INDEX RANGE SCAN
    - Nested Loop → Hash Join or NL 유지
    - FILTER 제거, Access Predicate로 변경

1. **[정리 및 마무리 – 5분]**
    - SQL 튜닝은 단순한 테크닉이 아니라,
        
        **문제를 해석하고, 전략을 수립하고, 결과를 증명하는 과정**이다.
        
    - 오늘 실습은 이 모든 단계를 한 문장 안에 녹여낸 실무형 훈련이다.
    - “튜닝은 단순히 SQL을 바꾸는 것이 아니다.
        
        **병목을 구조적으로 해결하고, 수치로 설명하는 일이다.**”
