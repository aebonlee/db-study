## 🕐 1교시: SQL 성능 진단 개요

**핵심 개념:**

- SQL 튜닝의 목적은 "SQL 실행 성능 향상"이 아니라 "병목 제거"
- Oracle SQL 처리 흐름: Parse → Bind → Execute → Fetch
- 튜닝 전, **전체 구조 진단(아키텍처/접근 방식)**이 우선되어야 함

**주요 키워드:**

SQL 처리 절차, Hard/Soft Parsing, 실행계획의 필요성, 튜닝 진단 루프

---

## 🕑 2교시: 실행계획 해석 기초

**핵심 개념:**

- 실행계획(EXPLAIN PLAN)은 SQL의 내부 처리 경로
- 하위 Operation부터 위로 실행됨 (Id 순서 = 실행 순서 아님)
- `TABLE ACCESS FULL`, `INDEX RANGE SCAN`, `FILTER`, `ACCESS` 등 구분

**주요 키워드:**

DBMS_XPLAN.DISPLAY, Rows, Cost, Access/Filter Predicate, 실행 방향

---

## 🕒 3교시: 조건절 튜닝 전략

**핵심 개념:**

- 조건절에 **함수, 산술 연산, LIKE '%X'** 포함 시 인덱스 무력화
- `OR` → `UNION ALL`, `NOT IN` → `NOT EXISTS`로 변환 권장
- WHERE절은 인덱스 활용 구조로 작성되어야 함

**주요 키워드:**

TO_CHAR 제거, IN vs EXISTS, OR 최적화, BETWEEN 권장

---

## 🕓 4교시: 인덱스 구조와 활용

**핵심 개념:**

- 인덱스는 B-tree 구조, 트리 깊이(BLEVEL)와 LEAF_BLOCKS 중요
- 단일 컬럼 인덱스보다 선행 컬럼 포함 다중 인덱스가 효과적
- CLUSTERING FACTOR가 낮을수록 인덱스 성능이 좋음

**주요 키워드:**

BLEVEL, LEAF_BLOCKS, RANGE SCAN, SKIP SCAN, CLUSTERING_FACTOR

---

## 🕔 5교시: 인덱스 설계 전략 및 조건절 연계

**핵심 개념:**

- 인덱스 활용은 조건절의 구조에 따라 결정됨
- 함수 기반 인덱스(Function-Based Index)로 조건절 함수 대응 가능
- 인덱스 설계 시, 조건절 사용 빈도, 정렬 순서, 필터링 강도 고려

**주요 키워드:**

Function-Based Index, 복합 인덱스 설계 원칙, 선행 컬럼 조건

---

## 🕕 6교시: 실행계획 실습 I

**핵심 개념:**

- DBMS_XPLAN 사용법 익히기
- 실행계획에서 **COST**, **ROWS**, **BYTES** 수치 비교
- Full Scan vs Index Scan 비교 실습을 통해 차이 체감

**주요 키워드:**

AUTOTRACE, COST 비교, 통계 수집 후 실행계획 변화 확인

---

## 🕖 7교시: 구조 튜닝 실습

**핵심 개념:**

- 조건절 변경 → 실행계획 변화 확인
- 인덱스 생성 및 제거 시 실행경로 변경 확인
- 실행계획 중심의 개선 시나리오 실습

**주요 키워드:**

DBMS_STATS.GATHER, 조건절 리팩토링, 실행계획 전후 비교

---

## 🕗 8교시: 종합 정리 및 실습

**핵심 개념:**

- 실행계획 해석, 조건절 최적화, 인덱스 적용 전략을 통합 실습
- 전후 성능 수치 비교로 성능 개선 효과 체감
- “튜닝은 실행 속도 향상이 아니라, 낭비 제거”라는 핵심 문장 이해

**주요 키워드:**

종합 튜닝 실습, 성능 비교표, 실행계획 해석력 점검

---
