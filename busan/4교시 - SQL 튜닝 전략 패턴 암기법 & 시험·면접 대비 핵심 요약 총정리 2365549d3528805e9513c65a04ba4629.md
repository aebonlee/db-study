# 4교시 - SQL 튜닝 전략 패턴 암기법 & 시험·면접 대비 핵심 요약 총정리

- 튜닝 상황에 맞는 패턴별 힌트, 조건절, 인덱스 적용 전략을 표 형식으로 정리할 수 있다.
- SQL 튜닝 면접/자격시험(예: SQLD, SQLP) 대비 키워드 기반 정리를 완성한다.
- SQL 튜닝을 말로 설명하거나 정리하는 능력을 강화한다.

1. **[조건절 기반 튜닝 전략 요약표 – 15분]**
    
    
    | 조건절 형태 | 문제점 | 개선 전략 |
    | --- | --- | --- |
    | `TO_CHAR(DATE) = '2022'` | 인덱스 무력화 | `BETWEEN TO_DATE()` 사용 |
    | `LIKE '%문자'` | Prefix Index 무력화 | Full Text Index 또는 구조 개선 |
    | `OR A = 1 OR B = 2` | Index Merge 또는 Full Scan | `UNION ALL`, `IN` 변경 |
    | `NOT IN`, `!=` | NULL 연산 문제 | `NOT EXISTS`, `ANTI JOIN` |
    | 함수 포함 서브쿼리 | 반복 호출 | JOIN으로 전환, INLINE MERGE |
    | 실행계획에 FILTER 표시 | 인덱스 미적용 | 조건절 구조 점검 |
    
    ✅ 암기 포인트:
    
    - TO_CHAR, OR, LIKE %, NOT IN = 튜닝 대상!
    - 구조가 아니라 조건부터 본다.

---

1. **[힌트 사용 요약 및 암기법 – 10분]**
    
    ▶ 가장 많이 쓰이는 힌트 5종:
    
    | 힌트 | 의미 |
    | --- | --- |
    | `INDEX(T)` | 특정 인덱스 강제 사용 |
    | `FULL(T)` | Full Scan 유도 |
    | `LEADING(A B)` | 조인 순서 고정 |
    | `USE_NL(T)` | Nested Loop Join 유도 |
    | `USE_HASH(T)` | Hash Join 유도 |
    
    📌 힌트 암기 문장 예:
    
    > "인덱스 줄까? 풀로 줄까? 리딩은 내가 정하고, 조인은 NL 아니면 HASH!"
    > 
    > 
    > 👉 패턴: **INDEX-FULL-LEADING-NL-HASH**
    > 

---

1. **[인덱스 적용 여부 판단표 정리 – 10분]**
    
    
    | 조건절 | 인덱스 사용 여부 |
    | --- | --- |
    | 컬럼 = 상수 | ✅ 사용 |
    | 컬럼 > 상수 | ✅ 사용 (단, Selectivity 낮으면 무시됨) |
    | 컬럼 + 1 = 값 | ❌ 함수 사용 → 무효 |
    | TO_CHAR(컬럼) = '값' | ❌ 무효 |
    | 컬럼 BETWEEN A AND B | ✅ 매우 효과적 |
    | SUBSTR(컬럼, 1, 2) = 'KO' | ❌ 무효, Function-Based Index 필요 |

1. **[SQL 튜닝 면접 대비 5문 5답] – 5분]**
    
    ① 인덱스가 있어도 Full Scan이 나오는 이유는?
    
    👉 조건절에 함수 사용, 통계 부정확, Selectivity 낮음
    
    ② Nested Loop vs Hash Join 튜닝 기준은?
    
    👉 소량 데이터 vs 대량 데이터
    
    ③ 바인드 변수 사용 이유는?
    
    👉 SQL 공유율 향상 → Soft Parsing 유도
    
    ④ 실행계획에서 FILTER가 의미하는 바는?
    
    👉 인덱스 미사용, 나중에 Row-Level 조건 적용
    
    ⑤ 히스토그램이 필요한 상황은?
    
    👉 특정 값의 데이터 편중이 심할 때
