# Day03 - 실습환경 설정하기

## ✅ Oracle 23ai Free 환경에서 CDB → PDB 접속 문제 해결 체크리스트

### 1. 기본 개념 정리

| 개념 | 설명 |
| --- | --- |
| **CDB** | Container Database. root 컨테이너 (`CDB$ROOT`) |
| **PDB** | 실제 사용되는 Pluggable Database (예: `FREEPDB1`) |
| **ORA-01017** | 사용자명/비밀번호가 잘못되었거나, 접속 대상 DB가 설정과 달라 발생 가능 |
| **ORA-65096** | CDB$ROOT에서 로컬 사용자 생성을 시도했기 때문에 발생. PDB 접속 후 시도해야 해결됨 |

---

### 2. 공통 설정 체크

### 📌 A. PDB 명 확인

```sql
-- SQLPLUS 또는 SQL Developer에서 실행
SHOW PDBS;
```

> 보통 FREEPDB1 또는 PDB1 등이 나옵니다.
> 

### 📌 B. PDB Open 상태 확인

```sql
ALTER PLUGGABLE DATABASE ALL OPEN;
```

### 3. SQL Developer에서 PDB로 접속 설정

### ✅ `접속 정보 설정`

- **호스트명**: `localhost` 또는 해당 IP
- **포트**: `1521`
- **서비스명**: `FREEPDB1` 또는 확인한 PDB 이름
- **사용자**: 예: `SCOTT` 또는 `HR`
- **비밀번호**: 맞게 설정
- **드라이버**: 기본 `Oracle JDBC Thin`

### ❗ `SERVICE_NAME` 명확히 기입

- `SID` 대신 **`SERVICE_NAME`** 사용: `FREEPDB1` (또는 tnsnames.ora 기준 값)
- TNS 사용 시 `EZCONNECT` 형태로도 가능:

```
localhost:1521/FREEPDB1
```

---

### 4. 사용자 계정 생성 주의사항

### ❌ CDB$ROOT에서 다음 명령어는 실패

```sql
CREATE USER scott IDENTIFIED BY welcome1;  -- ORA-65096 발생
```

### ✅ 해결 방법

```sql
-- 1. PDB 접속 후
ALTER SESSION SET CONTAINER=FREEPDB1;

-- 2. PDB에서 사용자 생성
CREATE USER scott IDENTIFIED BY welcome1;

-- 3. 권한 부여
GRANT CONNECT, RESOURCE TO scott;
```

### 5. SQL Developer 사용 중 ORA-01017 발생 시 확인사항

| 체크 항목 | 설명 |
| --- | --- |
| 사용자명 대소문자 | SQL Developer는 사용자명을 대소문자 구분할 수 있음 (`SCOTT` vs `scott`) |
| 비밀번호 오류 | 비밀번호 `welcome1` 정확히 입력되었는지 확인 |
| 접속 대상 | CDB인지 PDB인지 명확히 설정되었는지 확인 |
| 서비스명 | SID와 SERVICE_NAME 혼동하지 말고 `FREEPDB1` 명시 |

```
FREEPDB1 =
  (DESCRIPTION =
    (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    (CONNECT_DATA =
      (SERVICE_NAME = FREEPDB1)
    )
  )
```

> 🔧 경로 예시:
> 
> - Windows: `C:\Oracle\network\admin\tnsnames.ora`
> - macOS/Linux: `$ORACLE_HOME/network/admin/tnsnames.ora`

---

## ✅ 3. `listener.ora` 템플릿

```
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
    )
  )

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = FREEPDB1)
      (SID_NAME = FREEPDB1)
      (ORACLE_HOME = /opt/oracle/product/23ai/dbhomeXE)
    )
  )
```

> ⚠️ SID_NAME은 실제 PDB 이름(FREEPDB1)과 일치해야 하며,
> 
> 
> `ORACLE_HOME` 경로는 환경에 따라 수정 필요.
>

# ✅ Oracle PDB 실습 환경 구축 – 총괄 오류 정리 (SCOTT 계정 중심)

## 🎯 총 7대 오류 유형 및 해결 요약

| 구분 | 에러 코드 | 문제 요약 | 원인 | 해결 방법 |
| --- | --- | --- | --- | --- |
| 1️⃣ | `ORA-01017` | 비밀번호 불일치 또는 계정 없음 | 계정 없음 / 비밀번호 틀림 | 비밀번호 재설정 또는 SCOTT 계정 새로 생성 |
| 2️⃣ | `ORA-01035` | 접속 거부: RESTRICTED 모드 | PDB가 관리자 전용 제한 모드로 열림 | `ALTER PLUGGABLE DATABASE FREEPDB1 CLOSE IMMEDIATE;` 후 `OPEN;` |
| 3️⃣ | `ORA-01918` | 사용자 존재하지 않음 | SCOTT 계정이 PDB에 없음 | `CREATE USER scott IDENTIFIED BY welcome1 ...` 으로 새로 생성 |
| 4️⃣ | `ORA-01950` | USERS 테이블스페이스에 대한 할당량 없음 | QUOTA 없음 또는 테이블스페이스 없음 | `ALTER USER scott QUOTA UNLIMITED ON users;` 또는 `CREATE TABLESPACE users` |
| 5️⃣ | `ORA-01109` | 데이터베이스가 개방되지 않음 | PDB가 `MOUNTED` 상태 | `ALTER PLUGGABLE DATABASE FREEPDB1 OPEN;` |
| 6️⃣ | `ORA-01119` + O/S Error | 데이터파일 생성 실패 | 경로가 존재하지 않거나 권한 없음 | `DATAFILE` 경로를 `v$datafile` 참조하여 정확히 설정 |
| 7️⃣ | `ORA-12514` 또는 `ORA-12505` | 서비스 미등록 | 리스너가 `FREEPDB1` 인식 못함 | `ALTER SYSTEM REGISTER;`, `listener.ora`에 수동 등록 또는 `tnsnames.ora` 구성 |

---

## 📌 각 오류별 상세 가이드

### 1️⃣ `ORA-01017: 사용자명/비밀번호 불일치`

- **원인**: 잘못된 비밀번호 입력, 대소문자 오류, SCOTT 계정 없음
- **해결**:
    
    ```sql
    ALTER SESSION SET CONTAINER = FREEPDB1;
    ALTER USER scott IDENTIFIED BY welcome1;
    ALTER USER scott ACCOUNT UNLOCK;
    ```
    

---

### 2️⃣ `ORA-01035: RESTRICTED 모드`

- **원인**: PDB가 제한(RESTRICTED) 모드로 열림
- **해결**:
    
    ```sql
    ALTER PLUGGABLE DATABASE FREEPDB1 CLOSE IMMEDIATE;
    ALTER PLUGGABLE DATABASE FREEPDB1 OPEN;
    ```
    

---

### 3️⃣ `ORA-01918: 사용자 존재하지 않음`

- **원인**: SCOTT 계정이 PDB 내부에 없음
- **해결**:
    
    ```sql
    CREATE USER scott IDENTIFIED BY welcome1
    DEFAULT TABLESPACE users
    QUOTA UNLIMITED ON users;
    
    GRANT CONNECT, RESOURCE TO scott;
    ```
    

---

### 4️⃣ `ORA-01950: 테이블스페이스 할당량 없음`

- **원인**: QUOTA 미지정 또는 USERS 테이블스페이스 없음
- **해결**:
    - `테이블스페이스가 있을 경우`:
        
        ```sql
        ALTER USER scott QUOTA UNLIMITED ON users;
        ```
        
    - `테이블스페이스가 없을 경우`:
        
        ```sql
        CREATE TABLESPACE users
        DATAFILE 'C:\app\...\FREEPDB1\users01.dbf'
        SIZE 100M AUTOEXTEND ON;
        ```
        

---

### 5️⃣ `ORA-01109: 데이터베이스가 개방되지 않음`

- **원인**: `MOUNTED` 상태에서 접속 시도
- **해결**:
    
    ```sql
    ALTER PLUGGABLE DATABASE FREEPDB1 OPEN;
    ```
    

---

### 6️⃣ `ORA-01119` + `O/S Error: (OS 3)`:

- **원인**: 지정한 경로가 실제 존재하지 않음
- **해결**:
    - 경로 확인:
        
        ```sql
        SELECT file_name FROM dba_data_files;
        ```
        
    - 생성 명령:
        
        ```sql
        CREATE TABLESPACE users
        DATAFILE '정확한경로\users01.dbf'
        SIZE 100M AUTOEXTEND ON;
        ```
        

---

### 7️⃣ `ORA-12514` / `ORA-12505`: 서비스 인식 실패

- **원인**: 리스너가 `FREEPDB1`을 모름
- **해결 방법 3가지**:
    1. **자동 등록 유도**:
        
        ```sql
        ALTER SYSTEM SET LOCAL_LISTENER = '';
        ALTER SYSTEM REGISTER;
        ```
        
    2. **`listener.ora` 수동 등록**:
        
        ```
        SID_LIST_LISTENER =
          (SID_LIST =
            (SID_DESC =
              (GLOBAL_DBNAME = FREEPDB1)
              (SID_NAME = FREE)
              (ORACLE_HOME = ...)
            )
          )
        ```
        
    3. **`tnsnames.ora` 직접 지정**:
        
        ```
        FREEPDB1 =
          (DESCRIPTION =
            (ADDRESS = (PROTOCOL = TCP)(HOST = localhost)(PORT = 1521))
            (CONNECT_DATA = (SERVICE_NAME = FREEPDB1))
          )
        ```

# `ORA-12518` 에러

`ORA-12518: TNS:listener could not hand off client connection` 에러는 **Oracle Listener가 클라이언트의 연결 요청을 수락했지만 서버 프로세스에 할당하지 못했을 때** 발생하는 오류입니다. 주로 다음과 같은 이유로 발생합니다:

### ✅ **1. 서버 프로세스 부족 또는 제한**

- Oracle 인스턴스가 생성할 수 있는 서버 프로세스 수가 한계에 도달했을 때 발생합니다.
- 특히 **Windows** 환경에서 자주 발생합니다.

### 🔧 해결 방안

- `processes`, `sessions` 파라미터 확인 및 증가:

```sql
SHOW PARAMETER processes;
ALTER SYSTEM SET processes = 500 SCOPE=SPFILE;
ALTER SYSTEM SET sessions = 555 SCOPE=SPFILE; -- 일반적으로 processes * 1.1 + 5
```

- 변경 후 DB 재시작 필요.

---

### ✅ **2. 리스너와 인스턴스의 상태 불일치**

- Listener는 정상이나 DB 인스턴스가 DOWN 이거나 MOUNT 상태인 경우에도 발생합니다.

### 🔧 확인 방법

```bash
snrctl status
```

- Listener가 해당 인스턴스에 서비스 등록이 되었는지 확인.

### 🔧 해결 방안

- DB가 정상적으로 `OPEN` 상태인지 확인합니다.

```sql
SELECT status FROM v$instance;
```

- 필요시 인스턴스를 다음과 같이 오픈:

```sql
STARTUP
```

---

### ✅ **3. Dedicated 서버 프로세스 부족**

- 대부분 Dedicated 모드에서 발생하며, OS 자원 부족(특히 메모리, CPU)으로 인해 프로세스 생성 실패 시 발생.

### 🔧 해결 방안

- Shared Server 모드 전환 고려:

```sql
ALTER SYSTEM SET SHARED_SERVERS = 5;
ALTER SYSTEM SET DISPATCHERS = '(PROTOCOL=TCP)(SERVICE=ORCLXDB)';
```

---

### ✅ **4. Oracle Trace 설정**

- 좀 더 자세한 원인을 확인하려면 Listener Trace를 활성화하여 로그를 분석할 수 있습니다:

```bash
# listener.ora에 추가
TRACE_LEVEL_LISTENER = 16
TRACE_FILE_LISTENER = listener_trace
TRACE_DIRECTORY_LISTENER = /u01/app/oracle/diag/tnslsnr/...

# 리스너 재시작
lsnrctl stop
lsnrctl start
```

---

### ✅ **5. OS 리소스 확인**

- OS 수준에서 프로세스 수 제한, 메모리 부족 등 자원 제약이 있는지 확인:

```bash
ulimit -a  # (Linux)
tasklist   # (Windows에서 프로세스 수 확인)
```

---

### ✅ 요약

| 원인 | 해결 방안 |
| --- | --- |
| 서버 프로세스 수 부족 | `processes`, `sessions` 파라미터 증가 후 재시작 |
| 인스턴스 상태 불안정 | DB가 `OPEN` 상태인지 확인 |
| 리스너-DB 연결 불일치 | Listener 상태 확인 및 리스타트 |
| OS 자원 한계 | OS 레벨에서 프로세스/메모리 자원 점검 |
| Shared Server 필요 | `shared_servers`, `dispatchers` 설정 |
