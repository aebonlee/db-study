# 6교시 - 히스토그램 기반 옵티마이저 조정 전략 – NDV, 데이터 분포, Selectivity 제어 실습

### 🎯 학습 목표

- 옵티마이저가 실행계획을 선택할 때 **데이터 분포와 NDV(고유값 수)**를 어떻게 활용하는지 이해한다.
- *히스토그램(Histogram)**의 개념과 종류, 생성 방식 및 적용 전략을 학습한다.
- 통계정보가 잘못되었을 때 발생하는 실행계획 오류를 진단하고, 히스토그램을 통해 조정할 수 있다.
- `DBMS_STATS`를 활용한 히스토그램 수집 실습과, 히스토그램 적용 전후 실행계획 변화 비교를 수행한다.

---

### 🧑‍🏫 강의 흐름 및 설명 내용

1. **[도입 – 5분]**
    
    “우리는 지금까지 인덱스를 만들고, 조건절을 정리하고, 힌트를 사용하면서 성능을 개선해 왔습니다.
    
    하지만 때때로 옵티마이저는 '잘못된 실행계획'을 선택하곤 합니다.
    
    그 이유는 ‘통계정보’가 실제 데이터 분포를 반영하지 않기 때문입니다.
    
    오늘은 그 핵심 도구인 **히스토그램**을 통해 옵티마이저의 판단 기준을 직접 제어하는 방법을 배웁니다.”
    

---

1. **[히스토그램 개념 및 작동 방식 – 10분]**
    - 히스토그램이란?
        
        → 컬럼 값의 **분포도(Selectivity)**를 세분화하여 옵티마이저에게 전달하는 구조
        
    - 히스토그램이 필요한 상황:
        - 특정 값만 과도하게 많은 경우 (예: `STATUS = 'Y'`가 전체의 90%)
        - 기본 통계만 있으면 ‘고르게 분포’로 간주되어 Index 무시 가능
    - NDV: Number of Distinct Values
        - NDV가 작고, 분포가 왜곡되어 있을수록 히스토그램 필요
    - 히스토그램 종류:
        - HEIGHT BALANCED (기본값)
        - FREQUENCY (11g 이후 권장)
        - TOP-FREQUENCY (12c 이후 디폴트)

---

1. **[히스토그램 실습 – 수집 전후 실행계획 비교] – 15분**
    
    🔸 Step 1: 테이블 준비
    
    ```sql
    CREATE TABLE CUST (
      CUST_ID NUMBER,
      STATUS VARCHAR2(1)
    );
    
    BEGIN
      FOR i IN 1..10000 LOOP
        INSERT INTO CUST VALUES (i, CASE WHEN MOD(i,10)=0 THEN 'N' ELSE 'Y' END);
      END LOOP;
      COMMIT;
    END;
    ```
    
    🔸 Step 2: 통계 수집 (히스토그램 없음)
    
    ```sql
    EXEC DBMS_STATS.GATHER_TABLE_STATS(OWNNAME => USER, TABNAME => 'CUST');
    ```
    
    🔸 Step 3: 실행계획 확인
    
    ```sql
    EXPLAIN PLAN FOR SELECT * FROM CUST WHERE STATUS = 'N';
    SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
    ```
    
    🔸 Step 4: 히스토그램 수집
    
    ```sql
    EXEC DBMS_STATS.GATHER_TABLE_STATS(
      OWNNAME => USER,
      TABNAME => 'CUST',
      METHOD_OPT => 'FOR COLUMNS SIZE 10 STATUS'
    );
    ```
    
    🔸 Step 5: 다시 실행계획 확인
    
    ➤ 변화 포인트:
    
    - 히스토그램 없을 땐 `TABLE ACCESS FULL`
    - 히스토그램 적용 후 `INDEX RANGE SCAN` 가능성 ↑
    - `Rows`, `COST` 값 변화 체크

---

1. **[히스토그램 전략별 사용 시기 정리 – 10분]**
    
    
    | 조건 | 적용 전략 |
    | --- | --- |
    | 분포가 고르다 | 히스토그램 불필요 |
    | 특정 값이 몰려 있다 (偏) | FREQUENCY 히스토그램 |
    | NDV가 작고 정렬이 잘 되어 있다 | HEIGHT BALANCED |
    | 조건절이 특정 값 위주로 검색된다 | 히스토그램 필수 |
    
    ➤ 히스토그램은 모든 컬럼에 생성하지 말고, **조건절에 자주 쓰이고 비대칭적 분포일 때만** 생성
    

---

1. **[옵티마이저 오작동 사례 설명 – 5분]**
    - 히스토그램 없이 옵티마이저가 ‘고르게 분포’라고 가정하면
        
        → 실제로는 1% 조건인데도 50% 조건인 것처럼 계획을 잡음
        
        → INDEX 무시 + Full Scan 발생
        
    - 히스토그램 적용 → Selectivity 정확하게 전달
        
        → 실행계획이 조건별로 다르게 결정
        

---

1. **[정리 및 마무리 – 5분]**
    - 히스토그램은 옵티마이저에게 **“데이터가 어떻게 생겼는지를 설명해 주는 도구”**
    - 튜닝은 조건절 바꾸는 것만이 아니라, **옵티마이저의 눈을 바꾸는 작업**
    - “히스토그램은 실행계획을 고치는 것이 아니라,
        
        실행계획이 **올바로 선택되게끔 도와주는 설계도다.**”
