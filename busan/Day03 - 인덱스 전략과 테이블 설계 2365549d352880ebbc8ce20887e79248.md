## 🕐 17교시: 조인 튜닝 심화 – 조인 순서와 OUTER JOIN 전략

**핵심 개념:**

- 옵티마이저는 **선행 필터링이 높은 테이블을 먼저 수행**
- OUTER JOIN은 옵티마이저가 조인 순서를 변경할 수 없음 → 주의
- 조인 순서 강제: `LEADING`, `USE_NL`, `USE_HASH`

**주요 키워드:**

Nested Loop, Hash Join, OUTER JOIN 제한, 힌트 활용 전략

---

## 🕑 18교시: 인덱스 튜닝 심화 – 다중 컬럼, 함수 기반, Skip Scan

**핵심 개념:**

- 다중 컬럼 인덱스는 **선행 컬럼 조건 포함 여부**가 핵심
- 함수 기반 인덱스로 조건절 내 함수 사용 대응
- 후행 컬럼만 조건 시 Index Skip Scan 발생 → 효율 낮음

**주요 키워드:**

Function-Based Index, Index Skip Scan, Composite Index 설계 원칙

---

## 🕒 19교시: 비효율 SQL 탐지 – V$SQL 및 AWR 기반 진단

**핵심 개념:**

- SQL 성능 지표: ELAPSED_TIME, CPU_TIME, BUFFER_GETS, EXECUTIONS
- V$SQL에서 상위 부하 SQL 추출 후 실행계획 확인
- 반복 호출 SQL은 바인드 변수 미사용 가능성 높음

**주요 키워드:**

V$SQL, V$SQLAREA, ELAPSED_TIME 기준 정렬, SQL_ID 기반 분석

---

## 🕓 20교시: 고난도 튜닝 종합 실습 I – 병목 진단 및 구조 개선

**핵심 개념:**

- 실무형 SQL의 병목 구조를 직접 분석하고 리팩토링 수행
- Subquery → JOIN, 조건절 함수 제거, 인덱스 추가 적용
- 개선 전후 실행계획, COST, BUFFER_GETS 수치 비교로 효과 검증

**주요 키워드:**

실전 SQL 해석, 리팩토링 전후 실행계획 비교, 성능 수치 정리

---

## 🕔 21교시: 실무 SQL 구조 해석 – 스칼라 서브쿼리와 반복 병목 제거

**핵심 개념:**

- 스칼라 서브쿼리는 행 수만큼 반복 실행 → JOIN으로 통합하는 게 유리
- CASE WHEN EXISTS 구조는 OUTER JOIN 또는 INLINE VIEW로 대체 가능
- 실무 로직 해석을 통한 구조적 리팩토링 필요

**주요 키워드:**

스칼라 서브쿼리, CASE → OUTER JOIN 전환, EXISTS 병목 회피

---

## 🕕 22교시: Cursor Sharing 전략 – FORCE, BIND, 공유 실패 분석

**핵심 개념:**

- CURSOR_SHARING 설정 (`EXACT`, `FORCE`)에 따라 공유 가능성 달라짐
- 바인드 변수는 재사용을 가능하게 하나 BIND_MISMATCH로 실패할 수 있음
- 공유 실패 원인은 `V$SQL_SHARED_CURSOR.REASON`으로 확인 가능

**주요 키워드:**

Cursor Sharing, Bind Peeking, CHILD CURSOR, 공유 실패 유형

---

## 🕖 23교시: AWR Top SQL 분석 – Buffer Gets, Executions 중심 분석

**핵심 개념:**

- Buffer Gets가 높고 Rows가 적으면 캐시 낭비 가능성
- Disk Reads ↑ → 인덱스 부재 가능성
- AWR → DBA_HIST_SQLSTAT → SQL_ID로 추적 가능

**주요 키워드:**

DBA_HIST_SQLSTAT, AWR 리포트 해석, SQL 리소스 소비 구조 해석

---

## 🕗 24교시: 튜닝 과제 실습 I – SQL 분석 및 개선 전략 보고서화

**핵심 개념:**

- 수강자별 실습 SQL에 대해 병목 원인 진단
- 실행계획 → 조건절 → 인덱스 전략 수립
- 개선 전후 수치를 비교하여 **튜닝 보고서 초안** 작성

**주요 키워드:**

SQL 개선 보고서 구조, 실행계획 Before/After, COST/GETS 비교
