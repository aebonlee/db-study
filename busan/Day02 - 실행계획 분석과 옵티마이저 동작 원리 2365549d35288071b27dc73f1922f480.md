## 🕘 9교시: 커서 공유 실패와 하드 파싱 원인

**핵심 개념:**

- Oracle은 SQL 문장을 공유하려고 하지만 **리터럴 SQL**이 많으면 **하드 파싱** 증가
- SQL 공유 실패는 **Library Cache 경합** 및 **CPU 낭비** 유발
- `V$SQL_SHARED_CURSOR` 뷰에서 공유 실패 원인을 추적 가능

**주요 키워드:**

CURSOR_SHARING, BIND_MISMATCH, CHILD CURSOR, Hard vs Soft Parsing

---

## 🕙 10교시: 바인드 변수 적용 및 커서 튜닝 실습

**핵심 개념:**

- 바인드 변수 사용 시 SQL이 재사용되며 Soft Parse 유도
- CURSOR_SHARING = FORCE 설정 시 강제 바인드 적용 가능
- 실행계획 공유 시 바인드 민감성(Bind Peeking)에 주의

**주요 키워드:**

바인드 변수 (:1), V$SQL, FORCE/SIMILAR/EXACT, SESSION_CACHED_CURSORS

---

## 🕚 11교시: Shared Pool 경합 진단

**핵심 개념:**

- SQL이 공유되지 않으면 **Library Cache Latch 경합** 발생
- Literal SQL 남용 시 커서 수 증가 → LRU Flushing, Invalid Child 증가
- 자주 사용하는 SQL은 `session_cached_cursors`로 재사용 성능 향상 가능

**주요 키워드:**

V$LIBRARYCACHE, SHARED_POOL_SIZE, LATCH_FREE, MUTEX_SLEEP

---

## 🕛 12교시: 경합 진단 실습 및 튜닝 가이드

**핵심 개념:**

- 커서 공유 실패 시 `REASON` 항목으로 원인 분석
- 히스토그램, 바인드 타입, NLS 설정 등도 공유 실패 유발 가능
- 커서 재사용률 향상은 **SQL 튜닝 못지않은 시스템 성능 기여**

**주요 키워드:**

V$SQL_SHARED_CURSOR, PARSE_CALLS, EXECUTIONS, 바인드 불일치 진단

---

## 🕐 13교시: 실전 튜닝 유형별 전략 총정리 I

**핵심 개념:**

- 문제 유형별 대표 전략을 정리해 튜닝 포인트로 연결
- 구조 분석 없이 힌트만 쓰는 튜닝은 단기 효과만 있음
- SQL 구조 → 조건절 → 통계 → 실행계획 → 튜닝 전략 수립 흐름 필요

**주요 키워드:**

조건절 함수, LIKE %, OR 조건, NOT IN, 통계 부정확성

---

## 🕑 14교시: 서브쿼리 튜닝 전략

**핵심 개념:**

- IN → EXISTS → JOIN 구조로 리팩토링 시 성능 개선 가능
- 스칼라 서브쿼리는 **Row 수만큼 반복 실행** → JOIN으로 통합
- INLINE VIEW의 MERGE/NO_MERGE 전략 이해 필요

**주요 키워드:**

Subquery Unnesting, INLINE MERGE, 스칼라 서브쿼리 제거, EXISTS

---

## 🕒 15교시: 대용량 처리 전략 – 파티셔닝과 샘플링

**핵심 개념:**

- 데이터량이 크면 **접근 범위 제한(Partition Pruning)**이 핵심
- 병렬 쿼리는 분산 처리로 성능 향상 but 자원 고갈 주의
- 통계 수집 시 `estimate_percent`로 효율적인 샘플링 수행

**주요 키워드:**

PARTITION BY RANGE, PARALLEL QUERY, DBMS_STATS, AUTO_SAMPLE_SIZE

---

## 🕓 16교시: AWR 리포트를 활용한 SQL 진단

**핵심 개념:**

- AWR은 SQL 성능 문제의 Before/After 비교와 병목 진단에 효과적
- Top SQL, Buffer Gets, Disk Reads, Elapsed Time 기준 분석
- Snapshot 간 비교로 SQL 개선 효과 검증 가능

**주요 키워드:**

AWR REPORT, SQL ordered by Elapsed Time, BUFFER_GETS, DBMS_WORKLOAD_REPOSITORY
