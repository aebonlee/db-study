# 5교시 - 실전 SQL 튜닝 유형별 접근 전략 – 대량 테이블, 조건절 비효율, 통계 오류 등

### 🎯 학습 목표

- 성능 문제가 발생한 SQL의 특징을 빠르게 분석하고, 공통적인 비효율 유형을 식별할 수 있다.
- 조건절 구조, 인덱스 미사용, 통계 누락 등 대표적인 실무 SQL 튜닝 포인트를 이해한다.
- SQL 튜닝 시 전체 쿼리보다 **데이터 접근 방식**과 **Row Source의 Cost** 중심으로 판단하는 전략을 배운다.
- 실습을 통해 실행계획 변화에 따라 성능이 어떻게 달라지는지 분석한다.

1. **[대표적인 SQL 성능 저하 유형 설명 – 10분]**
    
    
    | 유형 | 증상 | 원인 |
    | --- | --- | --- |
    | 인덱스 미사용 | Full Table Scan | 조건절 연산자 사용, 함수 적용 |
    | 통계 오류 | Index 사용 안 함 | 통계정보 오래됨, 히스토그램 없음 |
    | 조인 순서 불합리 | Nested Loop 실패 | Driving Table 선택 오류 |
    | 서브쿼리 병목 | Row-by-Row 처리 | Scalar Subquery 반복 실행 |
    | 조건절 비효율 | CPU 사용 과다 | OR 조건, NOT IN, LIKE ‘%aaa’ |
    
    ➤ 핵심은 “실행계획을 해석하고, 그 선택 이유를 옵티마이저 입장에서 추론”하는 것
    

---

1. **[비효율 조건절 튜닝 전략 – 10분]**
    - 인덱스를 회피하는 조건절 패턴:
        - `WHERE TO_CHAR(COL) = ‘20230101’` → 함수 적용 시 인덱스 무효화
        - `WHERE SAL * 12 > 50000` → 산술식 적용도 인덱스 무효화
        - `LIKE ‘%abc’` → 인덱스 비효율, `LIKE ‘abc%’`로 변경 필요
    
    🔧 튜닝 전략:
    
    - 조건절에서 **컬럼 단독**으로 사용되도록 구조 수정
    - 가능하다면 조건절 우측에 상수만 위치하도록 조정

---

1. **[실습 – 조건절 튜닝 전후 실행계획 비교 – 15분]**
    
    🔸 문제 SQL
    
    ```sql
    SELECT * FROM EMP WHERE TO_CHAR(HIREDATE, 'YYYY') = '1981';
    ```
    
    🔸 튜닝 SQL
    
    ```sql
    SELECT * FROM EMP WHERE HIREDATE BETWEEN TO_DATE('1981-01-01','YYYY-MM-DD') AND TO_DATE('1981-12-31','YYYY-MM-DD');
    ```
    
    🔸 실행계획 비교
    
    ```sql
    EXPLAIN PLAN FOR ...
    SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
    ```
    
    ➤ 힌트 없이 인덱스 사용 유무 차이 확인
    
    ➤ 조건절 변경만으로 Full Scan → Index Scan 전환
    

---

1. **[실습 – OR 조건의 IN 변경, NOT IN → NOT EXISTS 변경 – 5분]**
    - `WHERE DEPTNO = 10 OR DEPTNO = 20`
        
        → `WHERE DEPTNO IN (10, 20)` 더 효율적
        
    - `WHERE DEPTNO NOT IN (...)`
        
        → `WHERE NOT EXISTS (...)` 로 바꾸면 NULL 처리 문제 회피 가능
        

---

1. **[정리 및 마무리 – 5분]**
    - 실전 튜닝은 실행계획의 구조보다 **데이터 액세스 방식**과 **행 수 예측(Row Cardinality)**을 보는 것이 중요
    - 옵티마이저는 ‘계산’을 통해 계획을 결정한다. 우리는 그 계산 근거를 분석해야 한다
    - “SQL 튜닝은 디버깅이 아니라, 설계 오류를 바로잡는 아키텍처 작업이다.”

### 📝 과제

2. 통계 수집 전후, 실행계획이 달라지는 SQL을 예시로 작성하고 비교 분석
3. `NOT IN`, `NOT EXISTS`, `MINUS` 각각의 튜닝 예시를 작성하고 Row Count 및 COST 비교
