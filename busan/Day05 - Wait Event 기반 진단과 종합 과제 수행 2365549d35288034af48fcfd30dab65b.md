## 🕐 33교시: 병렬 처리 튜닝 전략 – Parallel Query 실습

**핵심 개념:**

- 대용량 SQL은 **병렬 실행(PQ)** 으로 성능 개선 가능
- PQ 구조: Query Coordinator(QC) + PX 서버
- 실행계획에서 PX BLOCK, PX SEND 등으로 병렬 처리 흐름 확인

**주요 키워드:**

PARALLEL 힌트, DOP 설정, PX COORDINATOR, V$PQ_TQSTAT

---

## 🕑 34교시: 종합 튜닝 실습 I – 구조 변경, 힌트 적용, 실행계획 개선

**핵심 개념:**

- 복잡한 실무 SQL에 대해 구조적 분석 → 조건절 변경 + 힌트 적용
- 실행계획 COST, GETS, ROWS 비교로 성능 개선 검증
- 단일 기술 적용보다 **복합 전략 조합(조건+인덱스+히트)**이 효과적

**주요 키워드:**

실전 SQL 리팩토링, 구조 해석 → 힌트 조합, 실행계획 수치 비교

---

## 🕒 35교시: 튜닝 리포트 작성 및 발표 실습

**핵심 개념:**

- SQL 튜닝 결과는 반드시 **보고서화 및 수치화**되어야 함
- 발표 시 핵심 구성: 문제 진단 → 전략 설계 → 실행계획 변화 → 효과 수치 비교
- 실무에서는 **튜닝 설명력 = 설득력**

**주요 키워드:**

튜닝 보고서 템플릿, 개선 전후 수치표, 발표 구성 요소

---

## 🕓 36교시: 전략 정리 및 면접 대비 요약

**핵심 개념:**

- 조건절/힌트/인덱스 등 핵심 전략을 유형별로 정리
- 면접 또는 시험 대비용 튜닝 패턴 정리표 작성
- “많이 아는 것보다 빠르게 떠올릴 수 있는 정리력”이 실력

**주요 키워드:**

튜닝 암기표, 조건절 최적화 공식, 5대 힌트 기억법(INDEX, FULL, LEADING, NL, HASH)

---

## 🕔 37교시: 실전 문제 풀이 I – 조건절, 실행계획, 힌트 중심

**핵심 개념:**

- 시험 스타일의 선택형 문제를 통해 조건절 오류, 힌트 효과 등을 점검
- FILTER Predicate, 바인드 변수, Index Scan 여부 등 출제
- 해설을 통해 "왜 틀렸는지" 중심으로 이해력 강화

**주요 키워드:**

시험 대비 지문형 튜닝 판단, TO_CHAR, LIKE %, CURSOR_SHARING

---

## 🕕 38교시: 실전 문제 풀이 II – 조인 방식 및 옵티마이저 해석

**핵심 개념:**

- 조인 순서, 옵티마이저 전략 해석, 실행계획 변화 중심의 응용 문제
- 실행계획 수치 (Cost/Rows) 비교 → 최적화 판단 기준 제시
- 실무/면접 모두에서 요구되는 조인 판단력 강화

**주요 키워드:**

Nested Loop vs Hash Join, FILTER vs ACCESS, LEADING 힌트 영향

---

## 🕖 39교시: 종합 실습 발표 – 보고서 발표 및 피드백

**핵심 개념:**

- 수강생이 작성한 SQL 튜닝 결과를 실행계획 + 수치 + 전략으로 발표
- 발표 구조: 진단 → 전략 → 실행계획 변화 → 성능 개선 설명
- 실무와 가장 유사한 튜닝 커뮤니케이션 경험

**주요 키워드:**

SQL 튜닝 발표, 실행계획 해석력 평가, 수치 기반 설득력

---

## 🕗 40교시: 수료 리캡 및 확장 전략

**핵심 개념:**

- SQL 튜닝 40시간 전체 흐름을 리마인드
- 수강생 스스로의 성장 여정 정리
- 향후 심화 과정(PL/SQL, AWR/ASH, 병렬/파티션 튜닝) 연계 안내
- 수료 및 피드백 시간

**주요 키워드:**

튜닝 전체 로드맵 복습, 실무 적용 전략, 수료 설문, 과정 마무리
