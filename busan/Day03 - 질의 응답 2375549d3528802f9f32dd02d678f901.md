## 오류 메시지:

```
ORA-00932: 표현식의 데이터 유형 DATE은(는) 필요한 데이터 유형 NUMBER과(와) 호환되지 않습니다.
```

이 오류는 아래 SQL 문에서 발생한 것입니다:

```sql
SELECT * FROM customer
WHERE created_at > (SELECT TRUNC(AVG(created_at)) FROM customer);
```

---

## ❗ 문제 원인

- `AVG()` 함수는 **숫자형(NUMBER)** 타입에 사용하는 집계 함수입니다.
- 하지만 여기서는 `created_at` 컬럼이 **DATE** 타입입니다.
- `AVG(created_at)`은 `DATE` 타입에서 지원되지 않거나, 내부적으로 `NUMBER`로 변환할 수 없어서 오류가 발생합니다.

---

## ✅ 해결 방법

### 방법 1: `created_at`을 `NUMBER` 또는 `DATE`로 명시 변환 후 `AVG` 적용

```sql
-- DATE를 TO_NUMBER 함수로 변환 (1970 기준 일수 기준)
SELECT *
FROM customer
WHERE created_at > (
  SELECT TO_DATE(TRUNC(AVG(created_at - TO_DATE('1970-01-01','YYYY-MM-DD'))) + TO_DATE('1970-01-01','YYYY-MM-DD'))
  FROM customer
);
```

> created_at - 기준일 ➝ 숫자형 (일수) → AVG → 다시 TO_DATE로 환산
> 

---

### 방법 2: `UNIX_TIMESTAMP` 또는 날짜를 `NUMTODSINTERVAL()` / `NUMTODS()`처럼 Oracle 내장 함수로 처리

→ 이 방법은 Oracle에서는 **날짜 간 차이 → 숫자**, 다시 → 날짜 로 변환하는 방식이 핵심입니다.

```sql
-- 예시: 기준 평균 날짜 이후 고객 검색
SELECT *
FROM customer
WHERE created_at > (
  SELECT TRUNC(MIN(created_at)) + AVG(created_at - MIN(created_at))
  FROM customer
);
```

> 이 방식은 created_at - MIN(created_at)이 숫자(일수)로 평가되므로, AVG() 사용 가능
> 

---

## 🧪 참고: 테스트 예제

```sql
-- 평균 가입일 계산
SELECT TRUNC(MIN(created_at)) + AVG(created_at - MIN(created_at)) AS avg_created_at
FROM customer;
```

---

## 💡 결론

- `AVG(created_at)`은 직접적으로 사용할 수 없습니다.
- 반드시 `DATE - DATE` ➝ `NUMBER`로 변환 후 `AVG()` 연산
- 다시 `+ MIN(created_at)` 혹은 `+ 기준일`로 `DATE` 변환해야 오류 없이 실행됩니다.
