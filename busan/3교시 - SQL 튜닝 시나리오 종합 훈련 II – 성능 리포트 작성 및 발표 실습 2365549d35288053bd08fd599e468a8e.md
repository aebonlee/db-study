# 3교시 - SQL 튜닝 시나리오 종합 훈련 II – 성능 리포트 작성 및 발표 실습

### 🎯 학습 목표

- SQL 튜닝 결과를 실행계획 + AWR/V$SQL 수치 기반으로 정리하고 리포트화할 수 있다.
- 튜닝 결과를 성능 지표 중심으로 설명하고, 실무 문서나 회의에서 활용할 수 있는 **SQL 성능 개선 리포트**를 작성할 수 있다.
- 발표 연습을 통해 SQL 성능 개선 내용을 **논리적이고 설득력 있게 설명**하는 능력을 강화한다.
- 튜닝 리포트 발표 과정에서 다양한 개선 전략과 분석 관점을 상호 공유한다.

---

### 🧑‍🏫 강의 흐름 및 설명 내용

1. **[도입 – 5분]**
    
    “실무에서 SQL을 튜닝한 후, ‘빠릅니다’라고 말하는 것만으로는 부족합니다.
    
    ‘왜 빨라졌는가’, ‘무엇을 고쳤는가’, ‘얼마만큼 개선되었는가’를 **수치로 정리하고 설명할 수 있어야** 실무에서 신뢰를 얻을 수 있습니다.
    
    오늘은 여러분이 직접 튜닝한 SQL에 대해 **리포트와 발표**를 통해 결과를 정리해보겠습니다.”
    

---

1. **[리포트 양식 공유 및 작성 가이드 – 10분]**
    
    ▶ SQL 튜닝 성능 리포트 예시 양식:
    
    ```markdown
    [SQL 튜닝 보고서]
    
    1. 대상 SQL 설명:
       - 사용 목적:
       - 실행 빈도:
       - 원본 SQL 구조 요약:
    
    2. 문제 원인 분석:
       - 실행계획 요약:
       - 병목 지점:
       - 주요 지표 (COST, GETS, ROWS, READS 등):
    
    3. 개선 전략:
       - 조건절 수정:
       - 인덱스 생성 또는 힌트 적용:
       - 구조 변경 포인트:
    
    4. 개선 결과 비교:
    
       | 항목 | 개선 전 | 개선 후 | 개선률 |
       |------|---------|---------|--------|
       | COST | 380     | 55      | 85% ↓ |
       | BUFFER GETS | 28,000 | 3,100 | 89% ↓ |
       | DISK READS  | 4,500  | 320   | 93% ↓ |
    
    5. 결론 및 의견:
       - 개선 효과 요약:
       - 실무 적용 가능성:
       - 향후 과제:
    ```
    

---

1. **[리포트 작성 실습 – 20분]**
    - 각자 또는 조별로 이전 시간에 튜닝한 SQL을 바탕으로 리포트 작성
    - `DBMS_XPLAN.DISPLAY_CURSOR` 또는 `V$SQL`에서 수치 확보
    - 개선 전후 수치 기반으로 COST, EXECUTIONS, BUFFER_GETS, DISK_READS 정리
    - PPT 또는 A4 문서 양식으로 작성

---

1. **[튜닝 효과 발표 실습 및 피드백 – 10분]**
    
    ▶ 발표 시간: 3분 내외 × 2~3개 조 (시간 상황에 따라 조정)
    
    ▶ 발표 내용:
    
    - SQL 구조 및 문제점
    - 개선 전략 요약
    - 수치 기반 결과 비교
    - 적용 가능한 전략 강조
    
    ▶ 피드백 예시:
    
    - “조건절 변경만으로 COST가 80% 이상 줄어든 점이 명확히 잘 드러났습니다.”
    - “Filter Predicate → Access Predicate로 바뀐 점을 근거로 잘 설명하셨습니다.”

---

1. **[정리 및 마무리 – 5분]**
    - 성능 개선은 SQL을 바꾸는 것이 아니라,
        
        **수치로 결과를 말할 수 있어야 완성**이다.
        
    - “튜닝은 기술이 아니라 설명력이다.
        
        좋은 튜닝은 실행계획보다 보고서가 먼저다.”
        

---

### 📝 과제

1. 튜닝한 SQL에 대한 성능 보고서 1장 제출 (A4, 자유 형식)
2. 개선 전후 실행계획 캡처 또는 SQL_ID 기반 비교 스크린샷 포함
3. 발표 내용 중 타 수강생의 튜닝 전략 중 인상 깊은 점 1건 요약
