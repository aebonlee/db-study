# 8교시 - SQL 튜닝 결산 과제 I – 프로젝트형 튜닝 과제 설계 및 실행계획 보고서 작성

### 🎯 학습 목표

- 학습자가 직접 튜닝 대상 SQL을 선정하고 성능 병목을 진단할 수 있다.
- 튜닝 전후 실행계획과 성능 지표를 비교하고, 개선된 SQL을 리팩토링할 수 있다.
- SQL 튜닝 전 과정(진단 → 분석 → 재설계 → 검증 → 보고)을 독립적으로 수행할 수 있다.
- 성능 개선 효과를 정량적으로 설명하는 튜닝 보고서를 작성할 수 있다.

### 🧑‍🏫 강의 흐름 및 설명 내용

1. **[도입 – 5분]**
    
    “40시간의 SQL 튜닝 수업을 통해 이론, 실습, 분석, 구조 변경, AWR 활용까지 전 과정을 학습하셨습니다.
    
    이제는 여러분이 직접 SQL 하나를 선택하고, 병목을 진단하고, 성능을 개선하고,
    
    그 결과를 수치로 정리하여 **튜닝 보고서**를 작성해 보는 실습을 하겠습니다.”

1. **[과제 안내 – 실습 주제 선택] – 5분**
    
    ▶ 과제 ①: 성능 저하 SQL 직접 설계 또는 제공된 예시 선택
    
    - 예시 1: 함수 + IN 서브쿼리 + 조건절 혼합
    - 예시 2: 다중 조인 + OUTER JOIN + 통계 누락
    - 예시 3: 공유 실패 유발 SQL
    
    ▶ 과제 ②: 실행계획 분석 + 병목 추적
    
    ▶ 과제 ③: 구조 변경 또는 인덱스 적용
    
    ▶ 과제 ④: 실행계획 비교 및 보고서화
    

---

1. **[작업 가이드 – 튜닝 단계별 템플릿 제공] – 10분**
    
    📄 [SQL 튜닝 프로젝트 템플릿]
    
    - 📌 대상 SQL 설명
        - SQL 내용, 목적, 데이터량
    - 📌 문제점 진단
        - 실행계획, COST, Buffer Gets, Disk Reads, 공유 실패 여부
    - 📌 개선 전략
        - 조건절 변경, 인덱스 생성/삭제, 힌트 사용, 뷰 병합 등
    - 📌 개선 결과
        - 실행계획 Before vs After
        - 주요 수치: COST, Elapsed Time, Gets, Rows, Executions
    - 📌 정리 의견
        - 개선 방향의 타당성, 유의점, 향후 적용 가능성

1. **[개인/조별 실습 수행 – 20분]**
    - Oracle 실습 환경 접속
    - SQL 작성 및 실행 → 문제점 진단
    - 개선 SQL 설계 → 인덱스 구성, 힌트 적용
    - `DBMS_XPLAN.DISPLAY` 또는 `AUTOTRACE`로 실행계획 비교
    - 튜닝 전후 수치 비교표 작성
    - 결과 내용 기반으로 보고서 초안 작성

1. **[결과 요약 공유 및 피드백 – 5분]**
    - 조별 대표 1인씩 개선 SQL과 주요 수치 발표
    - 강사 코멘트:
        - “이 튜닝에서 가장 큰 포인트는 ○○이며, 개선 전보다 **COST가 n% 감소했습니다.**”
        - “이 SQL은 바인드 변수 사용으로도 공유 효과를 볼 수 있습니다.”

1. **[정리 및 마무리 – 5분]**
    - SQL 튜닝은 단순한 기술이 아니라 **진단·해석·설계·검증의 종합 과정**이다.
    - 오늘 과제를 통해 실무에서도 **보고서 기반 SQL 성능 개선 커뮤니케이션**이 가능해야 한다.
    - “성능이 좋아지는 SQL은 바뀐 SQL이 아니다.
        
        성능이 설명 가능한 SQL이 진짜 튜닝된 SQL이다.”
