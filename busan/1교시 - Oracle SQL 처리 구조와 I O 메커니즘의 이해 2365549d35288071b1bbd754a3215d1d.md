- 소프트파싱과 하드파싱의 차이 및 바인드 변수의 중요성을 학습한다.

- 향후 SQL 튜닝의 기반이 되는 Oracle 내부 아키텍처 개념을 학습한다.

### 🧑‍🏫 강의 흐름 및 설명 내용

1. **[도입 – 5분]**
    
    “여러분은 지금까지 SQL을 작성하고 실행하면서 결과만 확인했을 수 있습니다.
    
    그러나 우리가 DB 성능을 튜닝하려면, ‘SQL이 Oracle 내부에서 어떤 흐름을 거쳐 처리되는지’를 정확히 이해해야 합니다.
    
    오늘은 Oracle SQL이 파싱되고 실행되는 과정, 그리고 그 과정에서 발생하는 I/O가 어떻게 DB 성능에 영향을 미치는지를 배우겠습니다.”
    
2. **[개념 설명 – 15분]**
    - SQL이 DBMS 내부에서 처리될 때, 가장 먼저 거치는 과정은 ‘파싱’입니다.
        - *Hard Parsing:* SQL이 처음 등장했을 때 발생. 실행계획을 새로 계산하고 공유 풀(Library Cache)에 저장합니다.
        - *Soft Parsing:* 동일한 SQL이 이전에 캐시되어 있어 그대로 재사용하는 경우. 이때 CPU 자원을 절약하고 속도 향상.
    - **바인드 변수**를 사용하지 않으면 Literal SQL로 인해 하드파싱이 반복되어 성능 저하 발생.
    - SQL → Parser → Query Transformer → Optimizer → 실행계획 생성 → Execution
        
        이 일련의 흐름을 'SQL 처리 파이프라인'이라고 부릅니다.
        
3. **[Oracle 아키텍처와 I/O 구조 설명 – 15분]**
    - Oracle은 SQL 실행 시 **DB Block 단위로 I/O를 수행**합니다. 모든 작업은 Block 단위이며, 하나의 행을 조회해도 최소 1Block Access 발생.
    - **Random I/O** vs **Sequential I/O**
        - Random I/O: 인덱스를 경유할 때 발생. Block 위치가 불규칙하므로 디스크 접근 시간이 큼.
        - Sequential I/O: Full Scan 또는 대량 데이터 일괄 접근 시 발생. 블록이 연속되어 접근 효율이 좋음.
    - **Buffer Cache**를 통해 디스크 I/O를 줄이는 전략 이해
        - SQL이 접근한 블록이 메모리에 있으면 Logical I/O → 성능 향상
        - 없으면 Physical I/O 발생 → 속도 저하
4. **[실습 안내 및 환경 세팅 – 10분]**
    - 오라클 23 AI Free 버전의 SQL Developer 또는 웹 UI 환경에서 아래 쿼리를 실행하여 Block 단위 I/O를 확인하는 기초 실습 진행:
        
        ```sql
        SELECT * FROM EMP WHERE EMPNO = 7369;
        ```
        
        → 이 쿼리는 인덱스 조건이므로 Random I/O 발생
        
        다음 쿼리를 실행하여 Full Scan 확인:
        
        ```sql
        SELECT * FROM EMP;
        ```
        
        → Full Scan + Sequential I/O 발생. 실행계획 비교 필수.
        
    - 실행계획 확인:
        
        ```sql
        EXPLAIN PLAN FOR SELECT * FROM EMP WHERE EMPNO = 7369;
        SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
        ```
        
5. **[마무리 및 개념 정리 – 5분]**

    - 향후 학습할 내용들은 모두 오늘 배운 구조 위에서 튜닝을 진행합니다.
    - “SQL은 단순 명령어가 아니라, 오라클이라는 거대한 기계의 지시서입니다. 그 지시가 얼마나 효율적인지를 이해하는 것이 튜닝의 시작입니다.”

### 📝 과제

- EMP 테이블을 대상으로 다양한 조건으로 SELECT 쿼리를 작성하고 실행계획을 확인해보기
    
    ```sql
    SELECT * FROM EMP WHERE JOB = 'MANAGER';
    SELECT * FROM EMP WHERE SAL BETWEEN 2000 AND 3000;
    ```
    
- 각 실행계획을 캡처하여 어떤 방식의 I/O가 발생했는지 구분 (Index Scan, Full Scan 등)
