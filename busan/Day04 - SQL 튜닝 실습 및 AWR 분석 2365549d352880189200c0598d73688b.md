## 🕐 25교시: SQL 튜닝 발표 실습 – 성능 개선 설명력 강화

**핵심 개념:**

- SQL 개선 효과는 실행계획뿐 아니라 **수치 기반 설명력**으로 완성
- COST, BUFFER GETS, DISK READS, EXECUTIONS 전후 비교 필요
- 발표를 통해 **튜닝 전략 설계와 성능 결과 보고 능력**을 강화

**주요 키워드:**

SQL 튜닝 보고서, 발표 구성, 개선 효과 수치화, 실행계획 비교

---

## 🕑 26교시: 튜닝 패턴 회고 – 유형별 전략 정리

**핵심 개념:**

- 자주 등장하는 **조건절 문제 유형**을 전략적으로 정리
- IN/EXISTS, LIKE %, OR, NOT IN → 튜닝 대상
- 힌트, 인덱스, 조건절 구조 등을 조합한 **패턴 기반 대응법**

**주요 키워드:**

패턴별 대응 전략표, 조건절 튜닝 요약, 힌트 암기 구문(INDEX/FULL/LEADING)

---

## 🕒 27교시: 실행계획 병목 해석법 – Id, Rows, Cost 분석

**핵심 개념:**

- 실행계획은 **하위 Operation부터 실행됨**
- 병목 진단 포인트: Id별 Cost 집중, Rows 급증, Filter 발생 위치
- FILTER vs ACCESS Predicate 구분은 인덱스 사용 여부의 핵심

**주요 키워드:**

DBMS_XPLAN, Id 순서, Rows 예측, 실행계획 수치 진단

---

## 🕓 28교시: 종합 리팩토링 실습 – 조건절 + 인덱스 + 힌트 적용

**핵심 개념:**

- 실무형 복잡 SQL을 **조건절 정리 + 인덱스 적용 + 힌트 사용**으로 리팩토링
- 실행계획 전후 비교를 통해 실제 COST 차이 확인
- 튜닝 적용 방식은 **코드 수정보다 구조 재설계에 가까움**

**주요 키워드:**

종합 리팩토링, 힌트 적용 전략, 조건절 리팩토링 예제

---

## 🕔 29교시: 실무 SQL 시나리오 분석 – 반복 SQL 리팩토링 전략

**핵심 개념:**

- 스칼라 서브쿼리, EXISTS 구조의 반복 호출은 병목 요인
- 반복 SQL은 JOIN 또는 INLINE VIEW로 통합해 처리
- SQL을 기능 단위로 나누어 구조적으로 재작성

**주요 키워드:**

스칼라 서브쿼리 → JOIN, IN → EXISTS, OUTER JOIN 리팩토링

---

## 🕕 30교시: 옵티마이저 제어 – 히스토그램, NDV, Selectivity 튜닝

**핵심 개념:**

- 옵티마이저는 히스토그램 없이 **데이터 분포를 균등하게 가정함**
- 특정 값 편중 시 히스토그램을 생성해 옵티마이저 유도
- 통계 수집: `DBMS_STATS` + `METHOD_OPT`

**주요 키워드:**

히스토그램 유형(FREQUENCY, HEIGHT BALANCED), NDV, Selectivity

---

## 🕖 31교시: Lock & Wait Event 분석 – 트랜잭션 튜닝과 병목 대기 해결

**핵심 개념:**

- SQL이 느린 것이 아니라, **락 대기로 멈춰 있는 경우**
- `enq: TX – row lock contention`, `buffer busy waits` 등 WAIT EVENT 진단
- V$SESSION, V$LOCK으로 BLOCKING SESSION 추적

**주요 키워드:**

TX WAIT, BLOCKING_SESSION, 트랜잭션 커밋 타이밍, V$SESSION_WAIT

---

## 🕗 32교시: 인덱스 유지보수 전략 – Rebuild, Gather, Drop 판단 기준

**핵심 개념:**

- 인덱스는 성능 향상 도구이자 DML 성능 저하 요인
- BLEVEL ↑, LEAF_BLOCKS ↑, 사용률 ↓ → Rebuild or Drop
- 통계 정보 갱신(GATHER_STATS)도 튜닝의 일부

**주요 키워드:**

BLEVEL, CLUSTERING_FACTOR, V$OBJECT_USAGE, INDEX MONITORING
