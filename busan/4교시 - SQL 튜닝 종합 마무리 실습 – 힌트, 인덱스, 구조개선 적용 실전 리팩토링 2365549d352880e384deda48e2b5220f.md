1. **[튜닝 대상 SQL 제공 – 5분]**
    
    ▶ 실습용 SQL 예시:
    
    ```sql
    SELECT E.ENAME, D.DNAME, S.GRADE
    FROM EMP E, DEPT D, SALGRADE S
    WHERE TO_CHAR(E.HIREDATE, 'YYYY') = '1981'
      AND E.SAL BETWEEN S.LOSAL AND S.HISAL
      AND E.DEPTNO = D.DEPTNO
      AND D.LOC IN (
        SELECT LOC FROM DEPT WHERE DEPTNO BETWEEN 10 AND 20
      )
      AND E.JOB LIKE '%CLERK';
    ```
    
    ▶ 주요 비효율 요소:
    
    - TO_CHAR 함수 → 인덱스 무력화
    - IN 서브쿼리 → 중복 DEPT 접근
    - LIKE '%xxx' → Index 미사용
    - 조인 순서 불명확 → 옵티마이저 의존

---

1. **[실습 – 실행계획 분석 및 병목 진단] – 15분**
    - `EXPLAIN PLAN FOR` 실행
    - `DBMS_XPLAN.DISPLAY` 또는 `AUTOTRACE`로 실행계획 출력
    - 병목 구간 식별:
        - Full Scan 발생 테이블
        - Filter Predicate / Access Predicate 구분
        - Row 증가 급증 위치
        - COST 집중 구간 확인
    
    ➤ 성능 지표: COST, BUFFER GETS, ELAPSED TIME, ROWS
    

---

1. **[리팩토링 적용 – 힌트 / 조건절 / 인덱스 전략 사용] – 15분**
    - 조건절 리팩토링:
        
        ```sql
        E.HIREDATE BETWEEN TO_DATE('1981-01-01','YYYY-MM-DD') AND TO_DATE('1981-12-31','YYYY-MM-DD')
        ```
        
    - IN 서브쿼리 → JOIN 전환
    - JOB LIKE 조건 제거 또는 `Function-Based Index` 활용
    - 힌트 적용 예:
        
        ```sql
        /*+ LEADING(E D S) USE_NL(S) USE_HASH(D) */
        ```
        
    - 인덱스 생성:
        
        ```sql
        CREATE INDEX IDX_EMP_HIRE_JOB ON EMP(HIREDATE, JOB);
        ```

1. **[Before / After 비교 및 성능 수치 작성] – 10분**
    
    ▶ 비교표 작성:
    
    | 항목 | 튜닝 전 | 튜닝 후 | 개선률 |
    | --- | --- | --- | --- |
    | COST | 360 | 42 | 88%↓ |
    | BUFFER GETS | 22,000 | 1,500 | 93%↓ |
    | DISK READS | 3,200 | 200 | 94%↓ |
    | ROWS | 800 | 800 | 동일 |
    
    ▶ 튜닝 근거 문장 작성:
    
    - “인덱스 미적용의 원인이었던 TO_CHAR 제거 → Index Range Scan 유도”
    - “서브쿼리 제거 후 JOIN 전환 → 중복 DEPT 접근 제거”
