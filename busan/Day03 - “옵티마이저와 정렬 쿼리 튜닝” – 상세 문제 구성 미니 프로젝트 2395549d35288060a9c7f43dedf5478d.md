### ✅ [문제 1] 정렬 쿼리 성능 분석 (Sort / PGA 튜닝)

### 💡 문제 설명

다음 SQL은 전체 주문 목록을 주문일 기준으로 정렬하여 출력한다.

현재 서버의 PGA가 부족해 Sort 작업 시 디스크(Temporary Tablespace)를 사용하고 있으며 성능이 저하되고 있다.

```sql
SELECT order_id, customer_id, order_date
FROM orders
ORDER BY order_date DESC;
```

### 🎯 수행 과제

1. 위 SQL을 기본 설정에서 실행하고 실행 시간 및 TEMP 사용 여부 확인
2. 다음 설정을 적용한 후 동일 쿼리를 다시 실행하고 성능을 비교하라:

```sql
ALTER SESSION SET WORKAREA_SIZE_POLICY = MANUAL;
ALTER SESSION SET SORT_AREA_SIZE = 524288; -- 512KB
```

1. 다시 아래 설정으로 PGA를 늘린 후 차이를 확인하라:

```sql
ALTER SESSION SET SORT_AREA_SIZE = 10485760; -- 10MB
```

### 📌  풀이 후 생각할 부분

- 세 가지 설정 각각에 대해 실행 시간, TEMP 사용 여부, Disk Reads 변화
- `V$SQL` 또는 `V$TEMPSEG_USAGE` 또는 `DBMS_XPLAN.DISPLAY_CURSOR` 결과 확인하기
- 최적의 설정이 무엇이며 그 이유는 무엇인지 설명해보기

### ✅ [문제 2] GROUP BY 성능 튜닝 및 힌트 실험

### 💡 문제 설명

아래 쿼리는 고객별 주문 건수를 집계하여 출력한다. 이때 옵티마이저는 기본적으로 `HASH GROUP BY` 또는 `SORT GROUP BY`를 자동 선택한다. 힌트를 통해 방식의 차이를 체험하고 성능을 비교하라.

```sql
SELECT customer_id, COUNT(*) AS order_count
FROM orders
GROUP BY customer_id;
```

### 🎯 수행 과제

1. 기본 실행계획 확인 (GROUP BY 방식 확인)
2. 아래 힌트를 각각 붙여 실행계획과 성능을 비교하라:

```sql
-- ① Sort Group By
SELECT /*+ USE_SORT_AGGREGATION */ customer_id, COUNT(*)
FROM orders GROUP BY customer_id;

-- ② Hash Group By
SELECT /*+ USE_HASH_AGGREGATION */ customer_id, COUNT(*)
FROM orders GROUP BY customer_id;
```

### 📌  풀이 후 생각할 부분

- 각 쿼리의 실행계획 요약
- Buffer Gets, Elapsed Time 비교
- 옵티마이저가 기본적으로 선택한 방식이 어떤 것인지 설명
- 데이터 양이 많아질 때 어느 방식이 더 적합한지 의견 제시

### ✅ [문제 3] 통계 수집 전후 실행 계획 비교

### 💡 문제 설명

현재 `orders` 테이블에 대해 통계가 오래된 상태이다. 통계 수집 전후 실행계획이 어떻게 변하는지 관찰하라.

### 🎯 수행 과제

1. 아래 쿼리를 실행하고 실행계획을 캡처

```sql
SELECT * FROM orders WHERE order_date BETWEEN SYSDATE - 30 AND SYSDATE;
```

1. 다음 명령으로 통계 수집 수행

```sql
EXEC DBMS_STATS.GATHER_TABLE_STATS(USER, 'ORDERS');
```

1. 동일 쿼리를 재실행하여 실행계획 비교

### 📌 풀이 후 생각할 부분

- 통계 수집 전/후 실행계획 파악하기
- Cost, Cardinality, Access Path 변화 요약해 보기
- 옵티마이저의 판단이 어떻게 달라졌는지 구분하여 생각해 보기

### ✅ [문제 4] 병렬 처리 힌트 실험

### 💡 문제 설명

다음 쿼리에 대해 병렬 처리를 적용해 성능이 개선되는지 테스트하라.

```sql
SELECT customer_id, COUNT(*)
FROM orders
GROUP BY customer_id;
```

### 🎯 수행 과제

1. 기본 실행 및 응답 시간 확인
2. 다음 힌트가 있는 쿼리를 실행하여 차이를 확인

```sql
SELECT /*+ PARALLEL(orders 4) */ customer_id, COUNT(*)
FROM orders
GROUP BY customer_id;
```

1. `DBMS_XPLAN.DISPLAY_CURSOR`로 `PX` 관련 단계 확인

### 📌  풀이 후 생각할 부분

- 병렬도 적용 전후 실행계획 및 응답 시간 비교
- 병렬 처리로 인한 CPU 사용량 증가 여부 확인
- 병렬도 조정이 필요한 경우 그 이유 설명
