## 📘 Buffer Cache와 SQL 성능의 관계

1. **Buffer Cache란?**
    - Oracle SGA 내에서 가장 중요한 메모리 영역.
    - 디스크 I/O를 줄이기 위해 최근 읽은 블록을 메모리에 보관.
    - 기본적으로 LRU 방식으로 관리됨.
2. **Logical Read vs Physical Read**
    - Logical Read: Buffer Cache에서 읽는 것
    - Physical Read: 디스크에서 읽는 것
3. **Buffer Cache Hit Ratio**
    - `(1 - (Physical Reads / Logical Reads)) * 100`
    - 90% 이상이 일반적이나, 항상 좋은 것은 아님.
4. **Buffer Cache 활용 시 유의점**
    - 자주 호출되는 악성 SQL이 Hit Ratio를 왜곡시킬 수 있음.
    - SQL 튜닝으로 Random I/O를 줄이고 효율적으로 메모리를 활용해야 함.

## 🧪 실습 1: Buffer Cache Hit Ratio 확인

```sql
-- 현재 Buffer Cache Hit Ratio 확인
SELECT name, value
FROM v$sysstat
WHERE name IN ('db block gets', 'consistent gets', 'physical reads');

-- 참고: Buffer Cache Hit Ratio = (db block gets + consistent gets - physical reads) / (db block gets + consistent gets)
```

## 🧪 실습 2: Random I/O 유도와 성능 비교

### 1. 테이블 생성 및 데이터 삽입

```sql
CREATE TABLE big_customer (
    id NUMBER PRIMARY KEY,
    name VARCHAR2(100),
    city VARCHAR2(50)
);

BEGIN
  FOR i IN 1..100000 LOOP
    INSERT INTO big_customer VALUES (i, '고객_' || i, '서울');
  END LOOP;
  COMMIT;
END;
```

### 2. 인덱스 생성

```sql
CREATE INDEX idx_big_customer_name ON big_customer(name);
```

### 3. 인덱스 사용 vs Full Scan 비교

```sql
-- 인덱스 사용 (Random I/O 발생)
SELECT /*+ INDEX(big_customer idx_big_customer_name) */ *
FROM big_customer WHERE name = '고객_50000';

-- Full Scan 사용 (Sequential I/O)
SELECT /*+ FULL(big_customer) */ *
FROM big_customer WHERE name = '고객_50000';
```
