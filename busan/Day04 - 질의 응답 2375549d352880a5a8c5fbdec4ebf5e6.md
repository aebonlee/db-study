# Day04 - 종합적 오류 내역 및 해결 방안 (✏ 작성자 : 이애본)

## ✅ 1. `FETCH FIRST n ROWS ONLY` 오류 분석 (Oracle 11g)

### 🔍 오류 원인

- `FETCH FIRST n ROWS ONLY` 구문은 **Oracle 12c 이상부터 지원되는 ANSI SQL 2008 표준 문법**입니다.
- Oracle 11g에서는 해당 구문을 인식하지 못하므로 구문 분석 단계에서 오류 발생.

### ✅ 해결 방법

- Oracle 11g에서는 다음과 같이 **인라인 뷰 + `ROWNUM`** 을 활용해야 함:

```sql
SELECT *
FROM (
    SELECT * FROM emp ORDER BY sal DESC
)
WHERE ROWNUM <= 2;
```

- 실행 계획상 정렬 후 ROWNUM 필터링이 되며, 12c 이후 버전의 `FETCH FIRST`와 **논리적으로 동일한 처리 결과**가 생성됩니다.

---

## ✅ 2. `ALTER SESSION SET SORT_AREA_SIZE=800M` 오류

### 🔍 오류 원인

- ORA-02248: `"ALTER SESSION 옵션이 부적합합니다"` 오류는 `SORT_AREA_SIZE`가 Oracle 11g/12c 이후에서는 비권장이며, 수동 설정 시 일부 환경에서는 제한될 수 있음을 의미함.
- 특히 `WORKAREA_SIZE_POLICY=AUTO`일 경우 해당 파라미터는 무시됩니다.

### ✅ 해결 방법

1. 우선 다음과 같이 `MANUAL`로 정책 설정 변경:

```sql
ALTER SESSION SET WORKAREA_SIZE_POLICY=MANUAL;
ALTER SESSION SET SORT_AREA_SIZE=800000; -- 단위는 bytes
```

1. 이후 다시 자동으로 되돌릴 경우:

```sql
ALTER SESSION SET WORKAREA_SIZE_POLICY=AUTO;
```

---

## ✅ 3. `ALTER SESSION SET CURRENT_SCHEMA = DMLTEST` 오류

### 🔍 오류 원인

- ORA-01435: `"사용자가 존재하지 않습니다"` 오류는 `DMLTEST`라는 사용자가 DB 내 존재하지 않을 때 발생.
- 이 명령은 단순히 현재 세션에서 사용할 **스키마를 전환**하는 것으로, 해당 스키마가 존재해야 정상 실행됨.

### ✅ 해결 방법

- 다음과 같이 먼저 사용자를 생성해야 함:

```sql
CREATE USER DMLTEST IDENTIFIED BY your_password;
GRANT CONNECT, RESOURCE TO DMLTEST;
```

그 후 `ALTER SESSION SET CURRENT_SCHEMA = DMLTEST;` 실행 가능.

---

## ✅ 4. 파티션 테이블 생성 오류

### 🔍 오류 원인

- ORA-00906 `"누락된 좌괄호"`는 **테이블 생성 시 컬럼 정의가 빠져 있어서** 발생한 오류.
- `CREATE TABLE` 구문은 반드시 먼저 컬럼 정의 후 `PARTITION BY` 절을 명시해야 함.

### ✅ 해결 방법

- 아래와 같이 테이블 컬럼 정의를 추가하여 해결:

```sql
DROP TABLE big_sales PURGE;
CREATE TABLE big_sales
 (
    sale_id     NUMBER,
    sale_dt     DATE,
    product_id  NUMBER,
    amount      NUMBER
)
PARTITION BY RANGE (sale_dt)
(
  PARTITION p_old  VALUES LESS THAN (DATE '2024-01-01'),
  PARTITION p2024  VALUES LESS THAN (DATE '2025-01-01'),
  PARTITION pmax   VALUES LESS THAN (MAXVALUE)
);
```
