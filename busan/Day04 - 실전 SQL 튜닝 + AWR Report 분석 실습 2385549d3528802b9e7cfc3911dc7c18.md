# 🎯 실전 SQL 튜닝 + AWR Report 분석 실습 구성

## 🧠 1. 학습 내용 요약

| 주제 | 내용 |
| --- | --- |
| SQL 튜닝의 목적 | 최소한의 리소스로 최대한의 성능 확보 (I/O 감소, CPU 사용량 감소, 응답시간 단축 등) |
| 주요 대상 | 비효율적 실행계획, Full Scan 남용, 인덱스 미활용, Join 방식 오류 등 |
| AWR 리포트의 역할 | 시스템 전반의 병목 구간(Wait Event, SQL, 세션 등)을 정량적으로 분석 |
| AWR 주요 분석 항목 | Top SQL, Wait Event, Buffer Cache Hit Ratio, I/O 관련 이벤트 등 |
| SQL 튜닝 전략 | 인덱스 활용, 옵티마이저 힌트 사용, 조인 순서/방식 조정, 통계 수집 등 |
| AWR 리포트 위치 | `SYSAUX` 테이블스페이스에 저장, `DBMS_WORKLOAD_REPOSITORY` 또는 `awrrpt.sql` 통해 생성 |

## 🧪 2. 실습 예제 순서 및 설명

### 🧩 [실습 1] 비효율 SQL 작성 및 AWR 리포트 생성

### 📌 목표:

- 고의로 비효율적인 SQL을 작성하고 AWR 리포트에 그 영향을 기록하여 분석.

```sql
-- 예제 테이블 생성
CREATE TABLE orders AS
SELECT level AS order_id,
       MOD(level, 1000) AS customer_id,
       TO_DATE('2020-01-01', 'YYYY-MM-DD') + MOD(level, 365) AS order_date,
       DBMS_RANDOM.VALUE(100, 5000) AS amount
FROM dual CONNECT BY level <= 100000;

-- 비효율 SQL (풀스캔 유도)
SELECT *
FROM orders
WHERE TO_CHAR(order_date, 'YYYY-MM') = '2020-05';
```

### ✅ 해설:

- `TO_CHAR()` 함수를 사용해 인덱스 활용 불가 → Full Table Scan 유도.
- `order_date` 컬럼에 인덱스를 생성해도 사용되지 않음.
- 이 상태에서 AWR 스냅샷을 수동 생성하여 성능 데이터를 확보.

```sql
-- AWR 스냅샷 수동 생성
EXEC DBMS_WORKLOAD_REPOSITORY.CREATE_SNAPSHOT();

-- 비효율 SQL 반복 실행 (세션부하 유도)
BEGIN
  FOR i IN 1..10000 LOOP
    EXECUTE IMMEDIATE 'SELECT * FROM orders WHERE TO_CHAR(order_date, ''YYYY-MM'') = ''2020-05''';
  END LOOP;
END;
/

-- 스냅샷 종료
EXEC DBMS_WORKLOAD_REPOSITORY.CREATE_SNAPSHOT();
```

### 📊 [실습 2] AWR 리포트 분석

### 📌 목표:

- AWR 리포트를 생성하여 주요 Wait Event, Top SQL 분석.

```bash
-- SQL*Plus에서 실행
@?/rdbms/admin/awrrpt.sql
```

### ✅ 분석 포인트:

- Top 5 Timed Events: `db file scattered read`, `CPU Time`, `buffer busy waits` 여부 확인
- Top SQL by Elapsed Time: 위의 TO_CHAR SQL이 상위권인지 확인
- Buffer Cache Hit Ratio, I/O Throughput 확인

### 🛠 [실습 3] SQL 튜닝 적용

### 📌 목표:

- 비효율적인 SQL을 효율적으로 수정하고 실행계획을 확인

```sql
-- 인덱스 생성 (함수 없이 사용 가능하도록)
CREATE INDEX idx_orders_date ON orders(order_date);

-- 튜닝된 SQL
SELECT *
FROM orders
WHERE order_date BETWEEN TO_DATE('2020-05-01', 'YYYY-MM-DD')
                     AND TO_DATE('2020-05-31', 'YYYY-MM-DD');
```

```sql
-- 실행계획 확인
EXPLAIN PLAN FOR
SELECT *
FROM orders
WHERE order_date BETWEEN TO_DATE('2020-05-01', 'YYYY-MM-DD')
                     AND TO_DATE('2020-05-31', 'YYYY-MM-DD');

SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);
```

### 📈 [실습 4] 튜닝 적용 후 성능 비교

### 📌 목표:

- 동일한 쿼리 반복 수행 후, 다시 AWR 생성 → 효과 확인

```sql
EXEC DBMS_WORKLOAD_REPOSITORY.CREATE_SNAPSHOT();

BEGIN
  FOR i IN 1..10000 LOOP
    EXECUTE IMMEDIATE 'SELECT * FROM orders WHERE order_date BETWEEN TO_DATE(''2020-05-01'', ''YYYY-MM-DD'') AND TO_DATE(''2020-05-31'', ''YYYY-MM-DD'')';
  END LOOP;
END;
/

EXEC DBMS_WORKLOAD_REPOSITORY.CREATE_SNAPSHOT();
```

### ✅ 해설:

- `db file scattered read` → `db file sequential read`로 변경되었는지 확인
- Elapsed Time 감소, Logical Read 증가 여부 확인
- Buffer Cache Hit Ratio 개선 여부 확인
