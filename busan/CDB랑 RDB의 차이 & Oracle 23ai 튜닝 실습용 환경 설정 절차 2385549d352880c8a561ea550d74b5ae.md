# CDB랑 RDB의 차이 & Oracle 23ai 튜닝 실습용 환경 설정 절차

### ✅ **CDB (Container Database)**

**도입 배경**: Oracle 12c부터 도입된 아키텍처로, 클라우드 환경 및 다중 테넌시를 지원하기 위한 구조입니다.

**특징**:

1. **컨테이너 기반 구조**:
    - 하나의 CDB 안에 여러 개의 **PDB (Pluggable Database)**를 담을 수 있음.
    - 논리적으로 독립된 DB를 하나의 인스턴스에서 운영 가능.
2. **구성 요소**:
    - **CDB$ROOT**: 전체 DB 인프라를 담당.
    - **SEED**: PDB 생성을 위한 템플릿 역할.
    - **PDBs**: 실제 사용자 데이터가 저장되는 논리 DB.
3. **장점**:
    - 멀티 테넌트 환경에 적합.
    - 자원 절약 및 관리 용이.
    - 하나의 패치로 모든 PDB에 적용 가능.
4. **관리**:
    - CDB 레벨에서 인스턴스 시작/정지, 백업/복구 가능.
    - PDB 레벨에서 사용자, 스키마, 데이터 개별 관리.
5. **적용 예시**: 클라우드 환경(AWS RDS), SaaS 구조에 적합.

---

### ✅ **RDB (Relational Database)**

**기본 개념**: 전통적인 관계형 데이터베이스 아키텍처.

**특징**:

1. **단일 데이터베이스 구조**:
    - 하나의 인스턴스에 하나의 DB가 존재.
    - 모든 스키마 및 사용자 데이터가 같은 DB 안에 존재.
2. **구성 요소**:
    - 데이터파일, 컨트롤파일, 로그파일 등으로 구성.
    - 전통적인 방식으로 사용자/스키마 관리.
3. **장점**:
    - 구조가 단순하고 익숙함.
    - 초기 학습과 관리가 용이.
4. **제약**:
    - 다중 고객 환경을 하나의 DB로 분리 운영하기 어려움.
    - 버전별 기능 확장이 제한적.

---

### 🔍 **Oracle 11g vs Oracle 23ai 기준**

| 항목 | Oracle 11g | Oracle 23ai (또는 12c 이상) |
| --- | --- | --- |
| CDB 지원 | ❌ 미지원 | ✅ 기본 제공 |
| 구조 | RDB 단일 구조 | CDB + PDB 멀티테넌트 |
| 장점 | 단순함, 익숙함 | 효율적 자원 활용, 다중 DB 관리 |
| 적용 대상 | 전통적 단일 시스템 | 클라우드, 다중 환경 관리 |

---

### 🧠 Aebon's Tip

1. **Oracle 11g에서는 모든 DB가 독립적 인스턴스로 운영**되어 자원 낭비가 발생할 수 있습니다.
2. **Oracle 23ai 이상에서는 하나의 인스턴스로 여러 DB(PDB)를 관리**하므로, 개발/운영/테스트 환경을 하나의 CDB로 통합 가능하여 효율적입니다.
3. 실습 시에는 Oracle XE 18c/23ai 환경에서 `Non-CDB`와 `CDB` 모드를 모두 경험해 보시는 것을 추천드립니다.

> cdb로 접속되어서 에러가 발생하니 pdb로 세션을 변경해서 실습할 수 있도록 그리고 scott 계정을 생성해서 튜닝 실습할 수 있게 환경 설정부터 체계적으로 순서대로 알려줘
> 

Oracle 23ai (또는 12c 이상) 환경에서 기본 CDB로 접속된 상태에서 **PDB로 세션 전환**하고, **SCOTT 계정을 생성하여 SQL 튜닝 실습**을 진행할 수 있도록 **환경 설정 절차**를 단계별로 안내드립니다.

---

## 🔧 Oracle 23ai 튜닝 실습용 환경 설정 절차 (CDB → PDB + SCOTT 계정)

---

### ✅ 1단계: 현재 접속 확인 (CDB 접속 상태인지 확인)

```sql
SELECT name, open_mode, con_id FROM v$database;
SHOW CON_NAME;
```

- 결과가 `CDB$ROOT`면 현재 CDB 루트에 접속된 상태입니다.

---

### ✅ 2단계: PDB 목록 확인 및 이름 확인

```sql
SHOW PDBS;
```

예시 결과:

| NAME | OPEN_MODE | RESTRICTED |
| --- | --- | --- |
| PDB$SEED | READ ONLY | NO |
| PDB1 | READ WRITE | NO |

> 여기서 실습할 PDB 이름이 PDB1이라면 이후 해당 PDB로 전환합니다.
> 

---

### ✅ 3단계: PDB 오픈 (필요 시)

```sql
ALTER PLUGGABLE DATABASE PDB1 OPEN;
```

> 이미 오픈되어 있다면 이 단계는 생략 가능합니다.
> 

---

### ✅ 4단계: PDB로 세션 전환

```sql
ALTER SESSION SET CONTAINER = PDB1;
```

- `SHOW CON_NAME;`으로 다시 확인하면 `PDB1`이어야 합니다.

---

### ✅ 5단계: SCOTT 계정 생성 (기본 테이블 포함)

```sql

-- 사용자 생성
CREATE USER scott IDENTIFIED BY tiger;

-- 권한 부여
GRANT CONNECT, RESOURCE TO scott;
GRANT UNLIMITED TABLESPACE TO scott;

-- SCOTT 스키마에 기본 EMP/DEPT 테이블 생성 (필요 시)
CONNECT scott/tiger

CREATE TABLE dept (
  deptno NUMBER(2) PRIMARY KEY,
  dname VARCHAR2(14),
  loc VARCHAR2(13)
);

CREATE TABLE emp (
  empno NUMBER(4) PRIMARY KEY,
  ename VARCHAR2(10),
  job VARCHAR2(9),
  mgr NUMBER(4),
  hiredate DATE,
  sal NUMBER(7,2),
  comm NUMBER(7,2),
  deptno NUMBER(2),
  FOREIGN KEY (deptno) REFERENCES dept(deptno)
);

-- 예시 데이터 삽입
INSERT INTO dept VALUES (10, 'ACCOUNTING', 'NEW YORK');
INSERT INTO dept VALUES (20, 'RESEARCH', 'DALLAS');
INSERT INTO dept VALUES (30, 'SALES', 'CHICAGO');
INSERT INTO dept VALUES (40, 'OPERATIONS', 'BOSTON');

INSERT INTO emp VALUES (7369, 'SMITH', 'CLERK', 7902, TO_DATE('17-12-1980','DD-MM-YYYY'), 800, NULL, 20);
INSERT INTO emp VALUES (7499, 'ALLEN', 'SALESMAN', 7698, TO_DATE('20-02-1981','DD-MM-YYYY'), 1600, 300, 30);
-- (생략 가능, 나머지 EMP 데이터는 필요 시 제공 가능)

COMMIT;
```

---

### ✅ 6단계: SQL 튜닝 실습 환경 점검

```sql

-- SCOTT 사용자로 접속
CONNECT scott/tiger@PDB1

-- 간단한 SELECT 실행
SELECT * FROM emp WHERE deptno = 30;

-- 실행계획 확인
EXPLAIN PLAN FOR SELECT * FROM emp WHERE deptno = 30;
SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY);

```

---

## 🧠 Aebon's Tip

1. **AWR은 CDB 레벨에서 수집되므로**, 실습 후 튜닝 효과를 분석할 때 `CDB$ROOT`로 접속하여 `AWR REPORT`를 생성하세요.
2. **PDB마다 독립적인 사용자(SCOTT 등)를 생성**해야 하며, `CDB$ROOT`에서 생성한 사용자는 PDB에서 보이지 않습니다.
3. **SQL 튜닝 실습 중 주의할 점**:
    - 테이블에 인덱스 추가 시 `테이블 스페이스` 할당 확인.
    - `Resource` 롤이 최신 버전에서는 제한적일 수 있어 `UNLIMITED TABLESPACE` 추가 권장.
