| 구분 | 권장 사양 |
| --- | --- |

## 2. 소프트웨어 다운로드 & 전달 방식

1. **사전에 인터넷이 되는 곳에서 아래 파일을 전부 내려받아** USB (또는 외장 SSD)에 저장
    - oracle-database-free-23c_WIN_X64.zip (≈ 4.4 GB)
    - sqldeveloper-23.x-with-jdk.zip (≈ 0.55 GB)
    - swingbench-win64-2025-04-17.zip (≈ 0.18 GB)
    - sqlcl-25.2.zip (≈ 0.03 GB) — 선택
2. USB 패키지 → 연수원 PC로 복사(외부망 차단 환경).
3. 강의 첫날 USB는 **수강생용과 강사용 2개**로 분리(실습 스크립트가 SYS 권한 포함).

- 미리 설치를 전부해주신다면 설치 부분은 캡쳐 화면으로 안내만 하고 바로 실습을 진행하겠습니다.

## 3. 설치 절차 상세 (수강생 PC 기준)

### 3-1. Oracle 23c Free 설치

```powershell
powershell
복사편집
# 압축 해제 후
setup.exe
# 설치 마법사에서
  • Global DB name  : ORCL
  • Password        : Oracle123!
  • Character Set   : AL32UTF8
  • Memory %        : 40 % (16 GB RAM 기준)
```

설치가 끝나면 `C:\oracle\product\23c\dbhomeFree`가 ORACLE_HOME 으로 설정됩니다.

### 3-2. 환경 변수 (자동) 및 추가 PATH 확인

```powershell
powershell
복사편집
setx ORACLE_HOME "C:\oracle\product\23c\dbhomeFree"
setx ORACLE_SID  ORCL
setx PATH "%PATH%;%ORACLE_HOME%\bin"
```

### 3-3. SQL Developer (Portable)

ZIP을 `C:\tools\sqldeveloper`에 풀고 `sqldeveloper64.exe`를 바로가기 등록.

> JDK 포함 버전을 쓰면 추가 설정 없이 실행됩니다. Oracle
> 

### 3-4. SwingBench & JDK

1. Temurin JDK 17 MSI 설치 후 재부팅.
2. `swingbench.zip` → `C:\tools\swingbench` 압축 해제.
3. `charbench.bat` 첫 실행 시 환경 변수 `JAVA_HOME` 확인.

### 3-5. SQLcl (선택)

`sql.exe`를 `C:\tools\sqlcl`에 두고 PATH에 추가.

## 4. 데이터·스크립트 구축

| 스크립트 | 목적 | 실행 계정 | 요약 |

## 5. 실습에 필요한 DB 파라미터

```sql
sql
복사편집
ALTER SYSTEM SET control_management_pack_access='DIAGNOSTIC+TUNING' SCOPE=SPFILE;
ALTER SYSTEM SET statistics_level = TYPICAL           SCOPE=BOTH;
ALTER SYSTEM SET timed_statistics = TRUE              SCOPE=BOTH;
ALTER SYSTEM SET memory_target = 6G                  SCOPE=SPFILE; -- 16 GB RAM 예시
```

재기동 후 AWR/SQL Monitor 실습 가능.

## 7. 강의 전날 체크리스트

- [ ]  28 대 PC 모두 Oracle Service, Listener 기동 확인 (`lsnrctl status`).
- [ ]  `sqlplus / as sysdba` 로 접속 테스트.
- [ ]  `@?/rdbms/admin/awrrpt.sql` 샘플 보고서 생성 확인.
- [ ]  SQL Developer → `ORCL_PDB` 접속 ·Explain Plan 동작 확인.
- [ ]  SwingBench → TPC-C 1분 부하 테스트 TPS ≥ 150 TPS (16 GB RAM 기준).
- [ ]  USB 내 `reset_lab.bat` (드롭 & 재생성) 동작 확인.

## 8. 추가 팁 (망 분리 대비)

1. **패치 적용 불가**: 23c Free 기본 버전으로 유지 — 실습 범위에는 영향 없음.
2. **교재 예제 중 APEX, OEM EM Express 언급** → 오프라인 실습에선 제외하고 화면 캡처로 설명.
3. **세션 트레이스 파일** 경로는 `%ORACLE_BASE%\diag\rdbms\orcl\orcl\trace` — USB로 복사 후 분석.

## 🎯 **시각적인 인터페이스**

### ✅ 1. **Oracle SQL Developer (무료, 공식 추천)**

- **제공처**: Oracle
- **장점**:
    - GUI 기반으로 쿼리 작성, 실행, 결과 확인 가능
    - 테이블 설계, 인덱스, 뷰, 시퀀스 등 시각적 확인 가능
    - 실행계획, 튜닝 도구 기본 포함
- **설치 필요**:
    - Java 포함 버전 권장
    - Oracle Client 없이도 JDBC Thin으로 접속 가능

---

### ✅ 2. **Toad for Oracle (상용, 실무에서 많이 사용)**

- **제공처**: Quest Software
- **장점**:
    - 강력한 튜닝 도구, 디버깅, 프로파일링 기능 탑재
    - 트랜잭션 제어와 데이터 조작 GUI 탭이 뛰어남
    - 실시간 성능 모니터링 및 AWR 분석 기능 제공
- **단점**:
    - 유료 (무료 평가판 사용 가능)
    - OCI Driver 기반 → Oracle Client 설치 필요

---

### ✅ 3. **Orange for Oracle (국산, Toad와 유사)**

- **제공처**: 웨어밸리
- **장점**:
    - Toad와 유사한 UI, 국산 환경에 최적화
    - 한글 지원이 잘 되어 있어 국내 기업에서 선호
- **설치 필요**: Oracle Client 필요 (OCI 기반)

---

### ✅ 4. **웹 기반 SQL 실습 환경 (무설치 방식)**

- **Oracle Live SQL**
    - https://livesql.oracle.com
    - Oracle 계정만 있으면 웹에서 SQL 실습 가능
    - 간단한 SQL 튜닝과 실행계획 확인도 가능
- **장점**: 설치 불필요, 어디서나 접속 가능
- **단점**: 성능 분석, 대용량 처리 실습엔 한계

---

### ✅ 5. **Oracle 23c Free + SQL Developer 구성 (추천 실습 조합)**

- **운영체제**: Windows, macOS, Linux
- **환경 구성 방법**:
    1. Oracle 23c Free 버전 설치
    2. SQL Developer 설치
    3. Oracle 접속 계정 생성 (예: SCOTT)
    4. 실습용 스키마 및 예제 데이터 import

---

### 🔧 실습 구성 팁

- **초기화 작업**: `CREATE TABLE`, `INSERT`, `ANALYZE`, `CREATE INDEX` 등을 SQL Developer에서 순차 실행
- **튜닝 실습**: SQL Developer에서 `Explain Plan`, `Autotrace`, `SQL Monitor` 기능 활용
- **데이터 양 많게**: 1만 건 이상의 더미 데이터 생성으로 Full Scan vs Index Scan 차이 명확히 비교
- **AWR/ASH 보고서 실습**: Oracle Enterprise 또는 Oracle Cloud에서 가능
